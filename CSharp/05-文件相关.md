# 文件相关

很多人刚接触 C# 文件操作时，最容易混在一起的是这几类问题：

* 路径应该用 `Path`、`Directory`、`File` 里的哪一个？
* 怎么拿程序当前目录、上级目录、指定文件夹路径？
* 什么时候用 `FileInfo` / `DirectoryInfo`，什么时候直接用静态方法？
* 打开文件对话框里的筛选器字符串到底怎么写？

如果先记一句总定义，可以先记这个：

**文件操作通常可以拆成三层：路径拼接用 `Path`，文件读写用 `File`，目录管理用 `Directory`。**

这篇笔记就按这条主线来讲：

1. 先分清路径、文件、目录三个层面；
2. 常见路径获取方式；
3. 文件与目录的常用 API；
4. `FileInfo` / `DirectoryInfo` 的定位；
5. 打开文件对话框的常见写法；
6. 实际开发里的注意事项。

## 一、先分清路径、文件、目录

### 1. Path 负责路径字符串处理

`Path` 更偏“字符串层”的路径处理。

它常做的事情包括：

* 拼接路径；
* 取文件名；
* 取扩展名；
* 取目录名；
* 组合相对路径和绝对路径。

例如：

```csharp
string filePath = Path.Combine(@"D:\Data", "test.txt");
string fileName = Path.GetFileName(filePath);
string extension = Path.GetExtension(filePath);
string directory = Path.GetDirectoryName(filePath)!;
```

### 2. File 负责文件操作

`File` 更偏“文件本身”的操作。

它常做的事情包括：

* 判断文件是否存在；
* 读取文本；
* 写入文本；
* 删除文件；
* 复制文件；
* 移动文件。

例如：

```csharp
if (File.Exists(filePath))
{
 string content = File.ReadAllText(filePath);
 Console.WriteLine(content);
}
```

### 3. Directory 负责目录操作

`Directory` 更偏“文件夹”的操作。

它常做的事情包括：

* 判断目录是否存在；
* 创建目录；
* 删除目录；
* 获取目录下文件；
* 获取目录下子目录。

例如：

```csharp
string folder = @"D:\Logs";

if (!Directory.Exists(folder))
{
 Directory.CreateDirectory(folder);
}
```

所以可以先记住：

* `Path`：处理路径字符串；
* `File`：处理文件；
* `Directory`：处理目录。

## 二、常见路径怎么拿

路径相关问题是最常见的第一步。

### 1. 拼路径优先用 Path.Combine

不要手工拼 `\`，优先用：

```csharp
string logPath = Path.Combine(AppContext.BaseDirectory, "Logs", "app.log");
```

这样比自己写：

```csharp
string logPath = AppContext.BaseDirectory + "\\Logs\\app.log";
```

更稳，也更不容易漏斜杠。

### 2. 当前程序目录怎么拿

常见写法：

```csharp
string baseDirectory = AppContext.BaseDirectory;
```

这在现代 .NET 项目里通常比依赖 UI 框架的启动路径更通用。

如果是 WinForms，也常见：

```csharp
string startupPath = Application.StartupPath;
```

但如果你的代码不想和 WinForms 强耦合，优先考虑 `AppContext.BaseDirectory`。

### 3. 上级目录怎么拿

原来很多人会手工截字符串，实际上更稳的是用 `DirectoryInfo` 或 `Path` 相关 API。

#### 第一种：DirectoryInfo 连续取 Parent

```csharp
DirectoryInfo di = new DirectoryInfo(AppContext.BaseDirectory);
string parentPath = di.Parent!.FullName;
string grandParentPath = di.Parent!.Parent!.FullName;
```

#### 第二种：相对路径配合 FullName

```csharp
DirectoryInfo di = new DirectoryInfo(Path.Combine(AppContext.BaseDirectory, "..", ".."));
string fullPath = di.FullName;
```

#### 第三种：不推荐的手工字符串截取

像下面这种方式虽然也能做，但不推荐作为默认方案：

```csharp
string path = AppContext.BaseDirectory;
int index = path.LastIndexOf("\\", StringComparison.Ordinal);
string parent = path.Substring(0, index);
```

原因是可读性差，也更容易出错。

### 4. 常用系统目录怎么拿

例如：

```csharp
string desktop = Environment.GetFolderPath(Environment.SpecialFolder.Desktop);
string documents = Environment.GetFolderPath(Environment.SpecialFolder.MyDocuments);
string appData = Environment.GetFolderPath(Environment.SpecialFolder.ApplicationData);
```

这比把路径写死在代码里更合适。

## 三、文件常用操作

### 1. 判断文件是否存在

```csharp
if (File.Exists(filePath))
{
 Console.WriteLine("文件存在");
}
```

### 2. 读取文本文件

```csharp
string content = File.ReadAllText(filePath);
```

如果是多行：

```csharp
string[] lines = File.ReadAllLines(filePath);
```

### 3. 写入文本文件

```csharp
File.WriteAllText(filePath, "Hello C#");
```

追加写入：

```csharp
File.AppendAllText(filePath, "\r\nNew Line");
```

### 4. 复制、移动、删除文件

```csharp
File.Copy(sourcePath, targetPath, overwrite: true);
File.Move(sourcePath, newPath);
File.Delete(targetPath);
```

### 5. 文件读写异常要有心理预期

文件操作很容易因为下面这些原因失败：

* 文件不存在；
* 路径不合法；
* 文件正在被占用；
* 没有权限；
* 目标目录不存在。

所以真实项目里，文件操作通常需要配合异常处理。

例如：

```csharp
try
{
 string content = File.ReadAllText(filePath);
 Console.WriteLine(content);
}
catch (IOException ex)
{
 Console.WriteLine(ex.Message);
}
catch (UnauthorizedAccessException ex)
{
 Console.WriteLine(ex.Message);
}
```

## 四、目录常用操作

### 1. 判断目录是否存在

```csharp
if (!Directory.Exists(folderPath))
{
 Directory.CreateDirectory(folderPath);
}
```

### 2. 获取目录下的文件

```csharp
string[] files = Directory.GetFiles(folderPath);
```

如果只想要某种类型的文件：

```csharp
string[] txtFiles = Directory.GetFiles(folderPath, "*.txt");
```

### 3. 获取目录下的子目录

```csharp
string[] directories = Directory.GetDirectories(folderPath);
```

### 4. 删除目录

```csharp
Directory.Delete(folderPath, recursive: true);
```

这里要注意：

* 默认删除非空目录会失败；
* `recursive: true` 会递归删除里面的所有内容，要谨慎使用。

## 五、FileInfo 和 DirectoryInfo 怎么理解

很多初学者会问：

既然已经有 `File` 和 `Directory`，为什么还有 `FileInfo` 和 `DirectoryInfo`？

### 1. File / Directory 更像静态工具类

例如：

```csharp
bool exists = File.Exists(filePath);
Directory.CreateDirectory(folderPath);
```

这种写法很直接，适合做一次性操作。

### 2. FileInfo / DirectoryInfo 更像面向对象封装

例如：

```csharp
FileInfo fileInfo = new FileInfo(filePath);
Console.WriteLine(fileInfo.Name);
Console.WriteLine(fileInfo.Length);
Console.WriteLine(fileInfo.Extension);
```

```csharp
DirectoryInfo directoryInfo = new DirectoryInfo(folderPath);
Console.WriteLine(directoryInfo.FullName);
Console.WriteLine(directoryInfo.Exists);
```

它们适合：

* 你想频繁围绕同一个文件或目录操作；
* 你希望同时拿属性和做操作；
* 你希望写法更偏对象风格。

### 3. 怎么选

可以先用这个经验：

* 一次性判断、读写、创建、删除，优先用 `File` / `Directory`；
* 围绕同一个路径频繁读取属性和执行操作，可以考虑 `FileInfo` / `DirectoryInfo`。

## 六、OpenFileDialog 和 SaveFileDialog

桌面程序里经常会遇到文件选择对话框。

### 1. OpenFileDialog 常见属性

最常见的几个属性包括：

* `InitialDirectory`：初始目录；
* `Filter`：文件筛选器；
* `RestoreDirectory`：关闭后是否恢复到原目录；
* `Multiselect`：是否允许多选。

示例：

```csharp
OpenFileDialog ofd = new OpenFileDialog();
ofd.InitialDirectory = Environment.GetFolderPath(Environment.SpecialFolder.Desktop);
ofd.RestoreDirectory = true;
ofd.Filter = "文本文件|*.txt|C#文件|*.cs|所有文件|*.*";

if (ofd.ShowDialog() == DialogResult.OK)
{
 string fileName = ofd.FileName;
}
```

### 2. Filter 字符串怎么写

筛选器的常见格式是：

```csharp
"显示名称|通配符|显示名称|通配符"
```

例如：

```csharp
ofd.Filter = "文本文件|*.txt|C#文件|*.cs|所有文件|*.*";
```

多个图片格式也可以这样写：

```csharp
ofd.Filter = "Image Files|*.bmp;*.jpg;*.gif|All Files|*.*";
```

或者写得更中文一点：

```csharp
ofd.Filter = "所有文件|*.*|bmp文件|*.bmp|jpg文件|*.jpg";
```

### 3. SaveFileDialog 常见思路

保存文件对话框和打开文件对话框的思路类似。

例如：

```csharp
SaveFileDialog sfd = new SaveFileDialog();
sfd.Filter = "文本文件|*.txt|所有文件|*.*";
sfd.DefaultExt = "txt";

if (sfd.ShowDialog() == DialogResult.OK)
{
 File.WriteAllText(sfd.FileName, "保存内容");
}
```

## 七、实际开发里的几个建议

### 1. 路径拼接不要手写斜杠

优先用 `Path.Combine()`。

### 2. 不要把系统路径写死

桌面、文档、应用数据目录这类路径，优先用 `Environment.GetFolderPath()`。

### 3. 读写文件前先想清楚编码和异常

很多“乱码”或“读不出来”的问题，不是 API 不会用，而是编码、占用、权限这些边界没处理好。

### 4. UI 项目里尽量不要让大文件读写阻塞界面

如果文件操作很重，应该考虑异步读写或后台处理，而不是直接堵住界面线程。

### 5. 旧写法能跑，不等于最推荐

比如原来常见的 `Application.StartupPath`、手工 `Substring()` 取父目录、手拼路径字符串，这些写法不是不能用，而是现代代码里通常有更清晰、更稳的替代方式。

## 八、总结

最后把这篇内容压成几句最重要的话。

### 1. 三个最基础的类型分工

* `Path`：处理路径字符串；
* `File`：处理文件；
* `Directory`：处理目录。

### 2. 路径怎么拿更稳

* 程序目录优先考虑 `AppContext.BaseDirectory`；
* 系统特殊目录优先考虑 `Environment.GetFolderPath()`；
* 上级目录优先考虑 `DirectoryInfo.Parent` 或 `Path.Combine()`。

### 3. 对话框里最常记住什么

**`Filter` 的格式就是：显示名称 | 通配符。**

### 4. 最后记一套判断框架

你可以把文件相关操作记成下面四句：

* 路径处理找 `Path`；
* 文件读写找 `File`；
* 目录管理找 `Directory`；
* 路径对象化管理找 `FileInfo` / `DirectoryInfo`。

如果这四句稳定了，日常大部分文件路径问题都能比较快地落到正确 API 上。
