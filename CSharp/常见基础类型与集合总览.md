# 常见基础类型与集合总览

很多人刚学 C# 时，最容易乱掉的不是语法，而是“类型该怎么选”：

* `int`、`double`、`decimal` 分别适合什么场景？
* `string` 为什么看起来像引用类型，但又有些“值”的感觉？
* 数组、`List<T>`、`Dictionary<TKey, TValue>`、`HashSet<T>` 到底怎么选？
* `var` 能不能乱用？
* 值类型和引用类型到底会影响什么？

如果先记一句结论，可以先记这个：

**类型选择的核心，不是背定义，而是先分清“存的是什么数据”“要怎么访问数据”“数据会不会变化”。**

这篇笔记不追求把所有类型讲细，而是先建立一张稳定的地图：

1. 先分清值类型和引用类型；
2. 常见基础类型怎么分工；
3. 字符串怎么理解；
4. 数组和常见集合怎么选；
5. 实际开发里有哪些高频判断。

## 一、先分清值类型和引用类型

### 1. 值类型是什么

值类型变量通常直接保存数据本身。

常见值类型包括：

* `int`
* `double`
* `bool`
* `char`
* `struct`
* `enum`
* `decimal`
* `DateTime`

例如：

```csharp
int a = 10;
int b = a;
b = 20;

Console.WriteLine(a); // 10
Console.WriteLine(b); // 20
```

这里 `b = a` 后，`b` 拿到的是一份自己的值。

### 2. 引用类型是什么

引用类型变量通常保存的是对象引用，而不是对象内容本身。

常见引用类型包括：

* `class`
* `string`
* 数组
* `List<T>`
* `Dictionary<TKey, TValue>`
* 委托

例如：

```csharp
List<int> list1 = new List<int> { 1, 2, 3 };
List<int> list2 = list1;
list2.Add(4);

Console.WriteLine(list1.Count); // 4
```

这里 `list1` 和 `list2` 指向的是同一个列表对象。

### 3. 为什么这个区分重要

因为它会影响：

* 赋值时到底是复制值还是共享对象；
* 方法传参时你改的是副本还是同一份对象；
* 性能和内存分配特征；
* 空引用问题会不会出现。

所以值类型 / 引用类型不是死知识点，它直接影响代码行为。

## 二、常见基础类型怎么分工

### 1. 整数类型

最常见的是：

* `int`
* `long`
* `short`
* `byte`

日常业务里最常用的是 `int`。

例如：

```csharp
int age = 18;
long totalCount = 3000000000;
```

一般可以先记：

* 普通整数优先 `int`；
* 范围明显更大时再考虑 `long`；
* `byte` 更适合字节数据而不是普通计数。

### 2. 浮点和小数类型

最常见的是：

* `float`
* `double`
* `decimal`

它们最重要的区别不是“都能存小数”，而是使用场景不同。

可以先记这个经验：

* 科学计算、普通小数计算常用 `double`；
* 金额、价格、财务数据优先 `decimal`。

例如：

```csharp
double x = 3.14159;
decimal price = 19.99m;
```

`decimal` 常用于金钱相关，是因为它更适合十进制精度要求高的场景。

### 3. bool

`bool` 只有两个值：

* `true`
* `false`

适合表达“是/否”“开/关”“成功/失败”。

例如：

```csharp
bool isSuccess = true;
bool isAdmin = false;
```

### 4. char

`char` 表示单个字符。

例如：

```csharp
char grade = 'A';
```

如果是一整段文本，用的是 `string`，不是多个 `char` 拼接来写业务代码。

### 5. DateTime

`DateTime` 常用于日期时间处理。

例如：

```csharp
DateTime now = DateTime.Now;
DateTime today = DateTime.Today;
```

它虽然不是“基础语法类型”，但在业务里出现频率很高，通常可以和常见值类型一起记。

### 6. enum

如果一个值只会在有限几个状态里变化，通常比起直接用数字或字符串，更适合用枚举。

例如：

```csharp
public enum OrderStatus
{
 Pending,
 Paid,
 Shipped,
 Completed
}
```

这样写比直接拿 `1`、`2`、`3` 表状态更清楚。

## 三、string 怎么理解

`string` 在 C# 里很特殊，因为它是引用类型，但使用体验又很像值。

### 1. string 是引用类型

从类型体系上说，`string` 是引用类型。

### 2. 但 string 是不可变的

例如：

```csharp
string text = "abc";
text += "def";
```

这并不是在原来的字符串对象上直接改内容，而是生成了新的字符串。

所以它虽然是引用类型，但业务上很多时候会表现得比较“安全”。

### 3. 什么时候要注意性能

如果是大量字符串拼接，例如循环里不断 `+=`，可能带来额外分配。

这时通常要考虑：

* `StringBuilder`

例如：

```csharp
StringBuilder sb = new StringBuilder();
sb.Append("A");
sb.Append("B");
string result = sb.ToString();
```

所以可以先记：

* 普通文本处理直接用 `string`；
* 高频拼接、大量拼接时考虑 `StringBuilder`。

## 四、数组和常见集合怎么选

这是最容易在实际开发中卡住的一块。

### 1. 数组

数组适合：

* 元素个数固定；
* 类型相同；
* 主要按索引访问。

例如：

```csharp
int[] numbers = { 1, 2, 3, 4, 5 };
Console.WriteLine(numbers[0]);
```

数组的长度通常不适合频繁变化。

### 2. `List<T>`

`List<T>` 是最常用的泛型集合之一。

适合：

* 元素个数不固定；
* 需要动态增加或删除；
* 仍然主要按索引访问。

例如：

```csharp
List<string> names = new List<string>();
names.Add("Tom");
names.Add("Jerry");
```

如果你不确定最终数量，又需要“像数组那样顺序存放”，通常先想到 `List<T>`。

### 3. `Dictionary<TKey, TValue>`

字典适合“通过键快速找值”。

例如：

```csharp
Dictionary<int, string> users = new Dictionary<int, string>();
users[1] = "Alice";
users[2] = "Bob";

Console.WriteLine(users[1]);
```

适合场景：

* 用 ID 查对象；
* 用名称查配置；
* 用键值对组织数据。

### 4. `HashSet<T>`

`HashSet<T>` 适合“只关心元素是否存在，并且不想重复”。

例如：

```csharp
HashSet<string> tags = new HashSet<string>();
tags.Add("C#");
tags.Add("C#");

Console.WriteLine(tags.Count); // 1
```

适合场景：

* 去重；
* 判断一个值是否已经出现过；
* 集合交集、并集这类操作。

### 5. `Queue<T>` 和 `Stack<T>`

如果数据结构本身有明确的进出规则，就不要硬拿 `List<T>` 模拟。

* `Queue<T>`：先进先出
* `Stack<T>`：后进先出

例如：

```csharp
Queue<string> queue = new Queue<string>();
queue.Enqueue("A");
queue.Enqueue("B");
string first = queue.Dequeue();
```

```csharp
Stack<string> stack = new Stack<string>();
stack.Push("A");
stack.Push("B");
string top = stack.Pop();
```

## 五、怎么快速选集合

可以先按“访问方式”来判断。

### 1. 主要按索引访问

优先考虑：

* 数组
* `List<T>`

### 2. 主要按键查值

优先考虑：

* `Dictionary<TKey, TValue>`

### 3. 主要想去重或判断是否存在

优先考虑：

* `HashSet<T>`

### 4. 数据天然有先进先出或后进先出规则

优先考虑：

* `Queue<T>`
* `Stack<T>`

所以集合选择最核心不是“哪个更高级”，而是：

**你的访问方式和约束是什么。**

## 六、var 应该怎么理解

很多人初学时会把 `var` 误解成“弱类型”。

其实不是。

### 1. var 不是动态类型

例如：

```csharp
var name = "Alice";
```

这里编译器仍然知道它是 `string`。

`var` 只是“让编译器根据右侧推断类型”。

### 2. 什么时候适合用 var

通常在下面这些场景里可读性还不错：

* 右边类型非常明显；
* 泛型类型名很长；
* 匿名对象。

例如：

```csharp
var numbers = new List<int>();
var person = new { Name = "Tom", Age = 20 };
```

### 3. 什么时候别滥用 var

如果右边看不出是什么类型，`var` 会让代码更难读。

例如：

```csharp
var result = GetData();
```

如果 `GetData()` 返回什么完全看不出来，那这种地方未必是好选择。

## 七、实际开发里的几个高频判断

### 1. 金额不要默认用 double

金额、单价、财务结果，优先先想 `decimal`。

### 2. 一组固定数量数据先想数组

如果长度固定，而且主要顺序访问，数组很自然。

### 3. 不确定数量又需要增删时先想 `List<T>`

这是最常见的默认集合之一。

### 4. 查找速度和键值映射很重要时先想 Dictionary

如果你总是在“拿一个键找一个对象”，字典通常更合适。

### 5. 状态值不要随手用魔法数字

能用 `enum` 表达状态时，通常比直接用数字更清楚。

### 6. 不要只背“值类型在栈上、引用类型在堆上”

这种说法对入门有帮助，但很容易被记死。

真正更值得记的是：

* 值类型更像复制数据本身；
* 引用类型更像多个变量共享同一个对象引用。

先把行为理解清楚，比死背内存口号更重要。

## 八、总结

最后把这篇内容压成几句最重要的话。

### 1. 先分大类

* 值类型：更关注值本身；
* 引用类型：更关注对象引用。

### 2. 常见基础类型怎么记

* 整数默认先想 `int`；
* 普通小数多用 `double`；
* 金额优先 `decimal`；
* 状态优先考虑 `enum`；
* 文本默认就是 `string`。

### 3. 常见集合怎么记

* 固定长度顺序数据：数组；
* 可变长度顺序数据：`List<T>`；
* 键值映射：`Dictionary<TKey, TValue>`；
* 去重和存在性判断：`HashSet<T>`；
* FIFO / LIFO：`Queue<T>` / `Stack<T>`。

### 4. 最后记一套判断框架

你可以把类型和集合选择记成下面四句：

* 先想存的是什么；
* 再想怎么访问；
* 再想数据会不会变化；
* 最后再决定具体类型。

如果这四句稳定了，大部分“这个类型该怎么选”的问题都会变得更好判断。
