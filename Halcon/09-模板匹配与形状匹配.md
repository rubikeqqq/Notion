# 模板匹配与形状匹配

在 Halcon 里，很多真正有工程价值的检测流程，第一步并不是直接测量、识别或判定，而是先回答一个更基础的问题：

目标到底在哪里。

模板匹配和形状匹配，解决的就是这个问题。

它们的核心价值不是“找到了一个像目标的区域”，而是：

给后续流程提供稳定的位置、角度，甚至尺度参考。

## 一、为什么定位通常是第一步

很多工业检测项目都不是在固定、理想、完全不动的图像条件下运行。

常见情况包括：

* 工件位置会偏；
* 工件角度会转；
* 拍摄时存在轻微姿态变化；
* 后面要测量、识别、取 ROI，但这些区域并不是全局固定的。

如果不先定位，后面的流程通常只能靠“写死坐标”。

这在 Demo 里可能能跑，但在现场通常很脆弱。

所以在工程里，一个高频思路就是：

先定位，再基于定位结果做后续检测。

## 二、模板匹配到底在找什么

很多初学者会把模板匹配理解成：

“找一块长得像模板的图。”

这只对了一半。

在工业视觉里，更重要的是模板匹配输出的结果通常包括：

* 目标位置；
* 目标角度；
* 匹配分数；
* 有时还包括尺度或多个候选结果。

也就是说，匹配结果不只是“找到了没”，而是“找到了之后，后续流程怎么跟过去”。

## 三、Halcon 里常见的匹配思路

Halcon 中最常见的匹配主线，通常可以先这样理解：

### 1. 灰度相关类匹配

这一类更偏基于灰度相似性。

适合：

* 目标纹理相对稳定；
* 灰度特征明显；
* 形变不大。

### 2. 形状匹配

这一类更偏基于轮廓和几何形状。

适合：

* 目标外形比较稳定；
* 灰度波动较大，但轮廓结构相对稳定；
* 更关心位置和角度，而不是局部灰度细节。

在工业项目里，形状匹配通常更常见，也更符合很多定位任务的习惯。

## 四、为什么 shape model 很重要

如果只从实用角度理解，shape model 的价值可以概括成一句话：

用一个训练好的目标形状，去图像里找相同或相似的目标姿态。

它通常适合解决：

* 工件定位；
* 姿态校正；
* 同一个零件在不同位置、不同角度下的查找；
* 为后续 ROI 跟随和坐标变换提供基准。

相比直接靠固定区域取检测位置，shape model 的鲁棒性通常更高。

## 五、一个基础的形状匹配流程怎么理解

典型流程大致可以理解为：

1. 先准备模板图像；
2. 选择合适的模板区域；
3. 创建 shape model；
4. 在运行图像里查找模型；
5. 获取匹配位置、角度和分数；
6. 把这些结果交给后续流程。

如果用 HDevelop 风格表示，大致会看到这类流程：

```halcon
read_image (TemplateImage, 'template.png')
create_shape_model (TemplateImage, 'auto', rad(-20), rad(40), 'auto', 'auto', 'use_polarity', 'auto', 'auto', ModelID)
read_image (Image, 'sample.png')
find_shape_model (Image, ModelID, rad(-20), rad(40), 0.5, 1, 0.5, 'least_squares', 0, 0.8, Row, Column, Angle, Score)
```

这里最关键的不是记住参数顺序，而是理解下面几件事：

* 模型先创建，再反复用；
* 匹配不是返回图像，而是返回位置和姿态结果；
* Score 不是越高越神秘，它本质上是相似度参考；
* 匹配输出通常是后续几何变换和 ROI 跟随的起点。

## 六、模板区域怎么选，比算子本身更重要

很多匹配不稳，根本原因不是 find_shape_model 不行，而是模板选得差。

好的模板区域通常有这些特点：

* 结构清楚；
* 特征独特；
* 不容易和背景或其他位置混淆；
* 既不是太小，也不是把大量无关区域一起包进去。

不好的模板通常有这些问题：

* 只截了一块很普通的边；
* 背景比目标特征还强；
* 模板里包含很多会变化的内容；
* 模板太大，拖慢速度还降低区分度。

所以匹配是否稳定，往往先看模板选区，而不是先怪参数。

## 七、匹配结果最重要的几个量

### 1. Row 和 Column

表示匹配到的位置。

### 2. Angle

表示匹配到的角度。

这是后续坐标跟随的关键。

### 3. Score

表示匹配结果的相似度参考。

它很重要，但不要把它理解成“唯一真理”。

实际项目里要结合：

* 结果是否落在合理区域；
* 角度是否合理；
* 多个结果之间是否混淆；
* 后续流程是否跟得上。

## 八、匹配不只是找一个结果

很多初学者默认：

图里就一个目标，匹配就一个结果。

但现场里经常会遇到：

* 多个相似目标；
* 背景里也有相似边缘；
* 一个图里有多个候选位置；
* 目标存在局部遮挡或特征缺失。

所以你要准备面对这些问题：

* 取第一个结果还是多个结果；
* 怎么按分数或位置进一步筛选；
* 匹配到多个目标时，后续流程要如何处理。

## 九、模板匹配最常见的工程用途

### 1. 给 ROI 跟随提供基准

先找出目标，再把后面的检测区域跟着走。

### 2. 为坐标变换提供姿态

匹配拿到角度和位置，后面就可以做仿射变换或局部对齐。

### 3. 做粗定位

有些场景不是直接靠匹配完成所有检测，而是先粗定位，再接测量或识别。

### 4. 替代纯固定坐标方案

这是它在工程里最常见的价值之一。

## 十、几个很常见的误区

### 1. 只看匹配分数，不看后续是否稳定

一个分数看起来还可以的结果，不代表它真的适合作为后续基准。

### 2. 模板区域选得太随意

这通常比参数调得不够更致命。

### 3. 匹配范围放得过大

搜索范围太大，速度慢、误匹配风险也会上升。

### 4. 期望匹配解决所有问题

匹配解决的是定位问题，不是所有缺陷、测量和识别问题。

## 十一、什么时候说明匹配流程比较合理

通常至少满足：

1. 多张样本上都能稳定找到正确位置；
2. 角度和位置输出对后续流程有用；
3. 模板区域具有足够区分度；
4. 搜索范围和匹配阈值不过度宽松；
5. 匹配结果和后续 ROI、变换、测量能顺畅衔接。

## 总结

1. 模板匹配和形状匹配解决的核心问题是定位，而不是单纯“找到了没”；
2. 在工业项目里，匹配结果最有价值的是位置、角度和后续流程的可跟随性；
3. 模板区域选择往往比参数细调更影响稳定性；
4. 当后续检测不适合写死坐标时，先定位再检测通常是更可靠的主线。
