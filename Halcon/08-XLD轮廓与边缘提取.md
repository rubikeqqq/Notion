# XLD 轮廓与边缘提取

在 Halcon 里，如果 region 解决的是“目标大概在哪一块”，那么 XLD 更常解决的是：

目标边界到底在哪里。

很多时候，真正影响测量精度、拟合精度和几何定位精度的，不是区域本身，而是轮廓和边缘表达。

这篇就专门讲 XLD 这条主线。

## 一、为什么会需要 XLD

如果只是做面积筛选、数量判断、粗略有无检测，region 往往已经够用了。

但一旦你开始关心下面这些问题，XLD 的重要性就会明显上来：

* 边缘位置要更准；
* 要拟合圆、线、椭圆等几何元素；
* 要做高精度测量；
* 区域边界太粗，不足以支撑精细分析；
* 想把目标表达成轮廓而不是面积块。

这就是为什么很多项目会从 region 主线进一步走到 XLD 主线。

## 二、XLD 可以先怎么理解

可以先把 XLD 理解成：

一种更偏轮廓、边界、亚像素表达的对象形式。

和 region 相比，它不强调“这片区域包含哪些像素”，而更强调：

“边界在哪里，轮廓长什么样。”

所以它更适合：

* 边缘定位；
* 轮廓拟合；
* 高精度测量；
* 曲线、直线、圆弧等几何分析。

## 三、XLD 通常从哪里来

### 1. 直接从边缘提取算子得到

最常见的是：

```halcon
edges_sub_pix (GrayImage, Edges, 'canny', 1.5, 20, 40)
```

这类方式适合直接从灰度图里找边缘。

### 2. 从 region 边界转换得到

例如：

```halcon
gen_contour_region_xld (Region, Contours, 'border')
```

这类方式适合：

* 你已经有了稳定区域；
* 只是想进一步拿到边界表达；
* 后续要做轮廓显示或基础拟合。

### 3. 从线、圆等拟合结果反向组织轮廓

在更复杂场景中，也可能先提轮廓、再做几何拟合、再把拟合结果用于显示或分析。

## 四、为什么 edges_sub_pix 这么常见

它很常见的根本原因是：

工业视觉里很多关键检测点，本质上都和边界精度有关。

例如：

* 孔边缘在哪；
* 产品外框边在哪里；
* 贴合边是否偏移；
* 直线有没有歪；
* 圆弧半径到底是多少。

如果这些任务只用 region 边界去做，精度和稳定性往往不够。

而亚像素边缘提取可以提供更适合几何分析的输入。

## 五、XLD 和 region 的区别到底在哪

可以先用一个非常实用的方式区分：

### region 更适合

* 做面积；
* 做数量；
* 做位置大致统计；
* 做形态筛选；
* 做大块缺陷分析。

### XLD 更适合

* 看边缘；
* 做轮廓拟合；
* 做高精度几何测量；
* 表达线、圆、弧这类边界结构；
* 为测量和拟合提供输入。

这不是绝对边界，但作为入门判断非常有用。

## 六、一个基础边缘提取示例

```halcon
read_image (Image, 'part.png')
rgb1_to_gray (Image, GrayImage)
gauss_filter (GrayImage, SmoothImage, 5)
edges_sub_pix (SmoothImage, Edges, 'canny', 1.5, 20, 40)
```

这条流程很典型：

1. 读图；
2. 转灰度；
3. 先轻量平滑；
4. 再提亚像素边缘。

为什么平滑常常在前面？

因为边缘提取对噪声通常比较敏感，适度平滑有助于稳定结果。

## 七、XLD 后面最常接什么

### 1. 轮廓筛选

不是所有边缘都需要，通常还要筛选长度、位置、方向等条件。

### 2. 几何拟合

例如：

* 拟合直线；
* 拟合圆；
* 拟合椭圆；
* 提取角点或关键轮廓段。

### 3. 测量与尺寸计算

很多高精度尺寸，最终都依赖轮廓或边缘结果，而不是纯 region。

### 4. 可视化叠加

XLD 很适合叠加到原图上做显示，因为它本质上就是轮廓表达。

## 八、XLD 在项目里最常见的几个价值

### 1. 让边界表达更精细

当 region 边界太粗时，XLD 通常更合适。

### 2. 适合几何对象分析

如果你的任务是：

* 找线；
* 找圆；
* 看轮廓是否闭合；
* 做位置或角度拟合；

XLD 路径通常更自然。

### 3. 更容易接高精度测量

尤其是尺寸、同心度、边缘偏差、外轮廓精度等任务。

## 九、几个常见误区

### 1. 以为 XLD 一定比 region 更高级

不一定。

如果任务只是简单区域筛选，XLD 反而可能增加复杂度。

### 2. 边缘一多就直接用

边缘提取常常会带出很多无关轮廓，后续筛选同样重要。

### 3. 不做预处理就直接提边缘

这在噪声图上很容易得到很乱的结果。

### 4. 只关注“提出来了没有”，不关注边缘是否稳定

如果边缘位置在不同样本上抖动很大，后面的测量和拟合都会被影响。

## 十、什么时候说明你该从 region 切到 XLD

可以参考下面几个信号：

1. 你关心的是边界位置，而不是区域面积；
2. 你需要拟合线、圆、弧这类几何元素；
3. 你发现 region 边界精度不够；
4. 后续测量结果对边界位置非常敏感；
5. 你需要更精细的可视化轮廓表达。

## 总结

1. XLD 是 Halcon 中处理轮廓、边缘和高精度几何信息的重要对象形式；
2. 它更适合边界分析、几何拟合和高精度测量，不是用来替代所有 region 流程；
3. edges_sub_pix 是最常见的边缘入口之一，但前处理和后筛选同样重要；
4. 当任务核心从“区域块”转向“边界精度”时，通常就应该认真考虑 XLD 路线。
