# 图像预处理基础

在 Halcon 里，很多检测流程真正的成败，不是由最后那一步匹配、测量、识别决定的，而是由前面的预处理决定的。

因为后面的很多能力，本质上都依赖一个前提：

输入图像足够稳定。

这篇就把 Halcon 中最常见的图像预处理思路先整理清楚。

## 一、为什么预处理这么重要

在真实项目里，原始图像往往不会直接处于“最适合检测”的状态。

常见问题包括：

* 光照不均；
* 背景干扰；
* 噪声点太多；
* 边缘发虚；
* 对比度不够；
* 局部阴影导致阈值不稳。

如果不先把这些问题尽量收敛，后面的区域提取、轮廓分析、匹配和测量都会跟着波动。

所以预处理的目标通常不是“让图看起来更好看”，而是：

让后续步骤更稳定。

## 二、预处理常见在流程里的什么位置

一个很典型的视觉流程通常会像这样：

1. 读取图像；
2. 转灰度或通道处理；
3. 预处理；
4. 分割区域或提取边缘；
5. 再做匹配、测量、识别或缺陷分析。

所以预处理通常是后续分析之前的一道稳定化工序。

## 三、最常见的预处理目标

可以先把预处理的目的归纳成几类：

### 1. 去噪

让随机噪声、小颗粒、局部毛刺少一些。

### 2. 平滑

让局部灰度变化不要太剧烈，方便后续阈值或边缘分析。

### 3. 增强对比

让目标和背景更容易区分。

### 4. 做亮度校正

减轻光照不均带来的影响。

### 5. 为后续对象表达做准备

例如：

* 为 region 阈值分割准备；
* 为 XLD 边缘提取准备；
* 为模板匹配提高稳定性；
* 为 OCR 提升清晰度。

## 四、几类高频预处理方式

### 1. 均值滤波

均值滤波适合做比较基础的平滑处理。

例如：

```halcon
mean_image (GrayImage, MeanImage, 5, 5)
```

它的特点通常是：

* 能平滑局部灰度；
* 能减轻一部分噪声；
* 但边缘也可能被一起抹平。

所以它适合基础降噪，但不适合所有场景都直接套。

### 2. 中值滤波

中值滤波对椒盐噪声这类离散噪点往往更有效。

例如：

```halcon
median_image (GrayImage, MedianImage, 'circle', 3, 'mirrored')
```

它常见的价值是：

* 去掉孤立噪点；
* 比某些平滑方法更容易保住边缘。

### 3. 高斯平滑

高斯平滑通常更适合很多自然的去噪和边缘前处理场景。

例如：

```halcon
gauss_filter (GrayImage, SmoothImage, 5)
```

它在很多边缘提取前都会出现。

### 4. 动态阈值前的背景校正

当光照不均比较明显时，常见思路不是直接固定阈值，而是先做背景估计，再用动态阈值。

例如：

* 先平滑出背景趋势；
* 再和原图比较；
* 再提取目标。

这类思路在亮度波动较大的现场很常见。

## 五、固定阈值和动态阈值怎么理解

虽然阈值分割本身不完全等于预处理，但它和预处理关系很紧。

### 1. 固定阈值

例如：

```halcon
threshold (GrayImage, Region, 120, 255)
```

适合：

* 光照稳定；
* 背景差异明确；
* 样本变化小。

优点是简单、快。

问题是：

一旦光照或材质波动较大，就容易失稳。

### 2. 动态阈值

例如：

```halcon
dyn_threshold (GrayImage, MeanImage, Region, 15, 'light')
```

它更像是在比较：

当前像素和局部背景之间的差异。

适合：

* 光照不均；
* 背景慢变化；
* 目标相对局部背景更容易定义。

## 六、增强不等于一味拉高对比

很多人做预处理时，第一反应是：

“把图调得更亮、更锐、更清楚。”

这不一定对。

预处理真正应该追求的是：

后续流程更稳。

例如有些时候：

* 把对比拉太高，噪声也会被一起放大；
* 锐化后边缘更乱，测量反而不稳；
* 局部增强后，动态阈值效果反而变差。

所以预处理不是“越强越好”，而是“够用就好”。

## 七、一个典型的预处理示例

下面给一个比较常见的区域提取前流程：

```halcon
read_image (Image, 'part.png')
rgb1_to_gray (Image, GrayImage)
median_image (GrayImage, MedianImage, 'circle', 3, 'mirrored')
mean_image (MedianImage, MeanImage, 15, 15)
dyn_threshold (MedianImage, MeanImage, Region, 12, 'light')
connection (Region, ConnectedRegions)
```

这里大概体现了下面几个思路：

1. 先转灰度；
2. 再去掉离散噪声；
3. 再估计背景；
4. 最后做动态阈值分割。

这类写法在工业现场比“直接一个固定 threshold”通常更稳一些。

## 八、预处理要怎么选，不要靠感觉

比较实用的判断方式通常是：

### 1. 先看目标和背景的差异来源

例如是：

* 灰度差；
* 边缘差；
* 颜色差；
* 纹理差；
* 局部亮斑或暗斑。

### 2. 再看干扰主要来自哪里

例如：

* 噪声；
* 反光；
* 阴影；
* 光照不均；
* 背景纹理。

### 3. 最后决定预处理链

例如：

* 去噪为主；
* 背景校正为主；
* 对比增强为主；
* 或者几种轻量组合。

也就是说：

预处理不是固定模板，而是围绕后续目标表达来设计。

## 九、几个很常见的误区

### 1. 上来就堆很多滤波

滤波链太长，图虽然变“干净”了，但目标边缘也可能一起变钝。

### 2. 参数只在单张图上调

这种流程在现场一换产品、一变光照，通常就会露问题。

### 3. 不分后续是 region 流还是 XLD 流

如果后面主要走边缘和轮廓，预处理重点和区域分割不一定一样。

### 4. 预处理和 ROI 设计分开看

实际上两者经常一起决定稳定性。

## 十、什么时候说明你的预处理已经比较合理

通常至少满足下面几点：

1. 多张样本上后续提取结果比较一致；
2. 关键参数不是特别敏感；
3. 处理后目标和背景差异更清楚；
4. 没有为了“看起来漂亮”而引入过度处理；
5. 你能说清每一步预处理是为后面哪一步服务的。

## 总结

1. 预处理的目的不是美化图像，而是稳定后续流程；
2. 均值、中值、高斯平滑、背景校正和动态阈值是最常见的基础组合；
3. 预处理链要围绕目标差异和干扰来源设计，而不是机械套模板；
4. 预处理选得对，后面的 region、XLD、匹配和测量都会容易很多。
