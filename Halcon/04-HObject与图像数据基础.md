# HObject 与图像数据基础

在 Halcon 里，很多人第一次真正卡住的地方，不是某个算子不会用，而是：

为什么同样看起来像“图像处理”，有时候输出的是 image，有时候是 region，有时候又是 XLD。

如果这一层没理顺，后面的阈值、轮廓、匹配、测量都会学得很散。

这篇就专门把 Halcon 里最基础的数据对象先梳理清楚。

## 一、为什么要先理解对象模型

Halcon 的很多处理流程，本质上不是在对“一个模糊的图像变量”连续操作，而是在不同对象之间切换：

* 图像对象；
* 区域对象；
* 轮廓对象；
* 元组参数；
* 各类句柄。

也就是说：

你不是只在“改图”，而是在不停切换不同表达方式。

这也是 Halcon 和很多普通图像处理入门认知不一样的地方。

## 二、HObject 可以先怎么理解

可以先把 HObject 理解成：

Halcon 中用于承载图像类对象的一种统一对象类型抽象。

在 C# 里你经常会看到：

```csharp
HObject ho_Image;
HObject ho_Region;
HObject ho_Contours;
```

它们变量类型都可以是 HObject，但语义并不一样。

这正是初学者最容易混淆的地方：

类型看起来一样，不代表内容一样。

## 三、Halcon 里最常见的几类对象

### 1. Image

image 表示图像本身。

它最常见的来源包括：

* 从文件读取；
* 从相机采集；
* 从前一步图像算子输出；
* 从通道变换或裁剪得到的新图像。

这类对象适合做：

* 灰度处理；
* 滤波；
* 增强；
* 阈值前处理；
* 匹配输入；
* 显示。

### 2. Region

region 表示区域，也就是一块被分割出来的像素集合。

它最常见的来源包括：

* threshold；
* dyn_threshold；
* regiongrowing；
* 从其他对象转换得到的区域。

这类对象适合做：

* 面积筛选；
* 连通域分析；
* 形态学操作；
* Blob 分析；
* 位置和形状统计。

### 3. XLD

XLD 通常表示轮廓或边缘等亚像素轮廓对象。

它的常见来源包括：

* edges_sub_pix；
* lines_gauss；
* 从区域边界转换；
* 轮廓提取类算子。

这类对象更适合：

* 边缘分析；
* 高精度轮廓定位；
* 几何拟合；
* 测量前的边界表达；
* 轮廓显示。

### 4. Tuple

tuple 不是图像对象，而是一类参数和结果容器。

它经常用来存：

* 数值参数；
* 字符串；
* 多个坐标；
* 匹配结果分数；
* 行列位置；
* 角度和尺寸。

很多时候，Halcon 的最终结果并不是一个图像对象，而是一组 tuple 数值。

### 5. Handle

handle 可以理解成一类“已经创建好的工具或模型句柄”。

例如：

* shape model 句柄；
* measure 句柄；
* OCR 句柄；
* 标定模型句柄。

它们通常是先创建，再反复使用。

## 四、为什么 image、region、XLD 不能混着理解

这是初学者最常见的问题之一。

例如：

* 预处理通常是对 image 做；
* 面积筛选通常是对 region 做；
* 轮廓拟合通常是对 XLD 做；
* 匹配输入一般还是 image；
* 测量结果可能最后变成 tuple。

也就是说：

同一条流程里，对象的表达方式会不断变化。

如果你把它们都混成“图像变量”，后面就很难理解为什么某些算子接不上。

## 五、一个最基础的数据流示例

下面这条流程很适合理解对象切换：

```halcon
read_image (Image, 'part.png')
threshold (Image, Region, 120, 255)
connection (Region, ConnectedRegions)
select_shape (ConnectedRegions, SelectedRegions, 'area', 'and', 500, 999999)
gen_contour_region_xld (SelectedRegions, Contours, 'border')
area_center (SelectedRegions, Area, Row, Column)
```

这段里对象切换大概是：

1. Image：原始输入图像；
2. Region：阈值分割后的区域；
3. ConnectedRegions / SelectedRegions：筛选后的区域集合；
4. Contours：从区域边界转换出的 XLD；
5. Area、Row、Column：最后得到的 tuple 数值结果。

如果你能把这一段看顺，后面的很多流程就会容易很多。

## 六、在 C# 项目里为什么经常看到 HObject

在 HalconDotNet 里，很多图像类对象会以 HObject 形式出现，这是为了统一管理和传递图像对象。

但在理解层面，不能只停留在“它是 HObject”。

更重要的是要继续追问：

* 这个 HObject 当前承载的是图像吗；
* 是 region 吗；
* 是轮廓吗；
* 这个算子期待的输入到底是哪一种。

如果这一步不分清，很多工程代码虽然类型能接上，语义却可能是错的。

## 七、对象流转为什么会直接影响流程设计

这不是纯理论问题，它直接决定你后面流程怎么搭。

例如：

### 1. 如果你做区域缺陷检测

通常主线会更偏：

image -> region -> region 筛选 -> 缺陷统计

### 2. 如果你做边缘和几何拟合

通常主线会更偏：

image -> XLD -> 轮廓分析 -> 几何结果

### 3. 如果你做模板匹配后测量

通常主线会更偏：

image -> matching -> pose / tuple -> 仿射变换 -> image / XLD / region 后续处理

所以，对象模型不是入门小知识，而是流程设计基础。

## 八、初学者最容易混淆的几个点

### 1. 把 region 当成图像

region 不是灰度图像本身，而是一个像素区域表达。

### 2. 以为 XLD 就是“画出来的线”

XLD 不只是显示线条，它本质上是轮廓对象，后续可以继续参与分析和拟合。

### 3. 只关注对象类型，不关注对象集合

很多 Halcon 对象不是单个目标，而是一组对象集合。

例如：

* 多个连通域；
* 多条轮廓；
* 多个匹配结果。

### 4. 忽略 tuple 的重要性

很多最终判定结果并不保存在图像对象里，而是在 tuple 数值里完成。

## 九、一个更贴近项目的理解方式

可以把 Halcon 里几类对象简单理解为：

* image：原材料；
* region：分割出的区域块；
* XLD：边界和轮廓表达；
* tuple：数值结果和参数；
* handle：可复用的工具或模型。

这个理解虽然简化，但足够支撑入门阶段的大多数场景。

## 总结

1. Halcon 的学习难点之一，不是算子多，而是对象表达方式多；
2. image、region、XLD、tuple、handle 这几类对象要尽早分清；
3. 很多流程本质上就是对象在不同表达形式之间切换；
4. 一旦对象模型理解顺了，后面的预处理、匹配、测量、识别都会更清晰。
