# 区域处理与连通域分析

在 Halcon 里，region 是最常见、也最容易最先上手的一类对象。

很多检测任务并不需要一开始就上模板匹配、复杂边缘分析或者高精度测量，只要能把目标区域稳定提出来，再做一些面积、形状和位置筛选，问题就已经解决了一大半。

这篇就专门讲 region 这条主线。

## 一、什么时候应该优先考虑 region

如果一个目标满足下面这些特点，通常可以先优先尝试 region 思路：

* 目标和背景有比较明显的灰度差；
* 目标区域大致连续；
* 最终关注的是面积、位置、长宽、圆度、矩形度等区域特征；
* 不一定需要亚像素级边缘精度；
* 更关心筛选和统计，而不是边界拟合本身。

例如：

* 缺料检测；
* 大面积脏污检测；
* 孔洞区域提取；
* 标签有无判断；
* 焊点区域筛选；
* 大致轮廓目标计数。

## 二、region 流程通常长什么样

一个典型的 region 流程通常可以理解成：

1. 输入图像；
2. 预处理；
3. 做阈值或分割；
4. 得到 region；
5. 做连通域分析；
6. 根据面积、形状、位置筛选；
7. 输出最终目标区域。

也就是说：

region 处理通常不是一步算子，而是一整条“分割 + 清洗 + 筛选”的流程。

## 三、最基础的 region 生成方式

### 1. threshold

最基础的区域提取方式通常就是阈值：

```halcon
threshold (GrayImage, Region, 120, 255)
```

它的含义是：

提取灰度在 120 到 255 之间的区域。

这适合：

* 目标亮于背景；
* 光照相对稳定；
* 灰度差足够明显。

### 2. dyn_threshold

如果光照不够稳定，常见就会转向动态阈值：

```halcon
dyn_threshold (GrayImage, MeanImage, Region, 15, 'light')
```

它更适合：

* 背景有亮度缓慢变化；
* 现场阴影或光照不均较明显；
* 需要按局部背景差异提区域。

## 四、为什么一定要理解连通域

很多时候阈值出来的 Region 并不是“一个目标”，而是一堆目标和噪声混在一起。

这时最关键的一步通常就是：

把连在一起或彼此分离的区域拆开来看。

最常见的算子就是：

```halcon
connection (Region, ConnectedRegions)
```

它的作用可以简单理解成：

把当前区域拆成一个个独立连通块。

这一步一出来，后面很多筛选就变得容易很多。

## 五、select_shape 为什么这么高频

在 region 流里，select_shape 基本是高频核心算子之一。

因为大多数项目真正想做的不是“把所有区域都提出来”，而是：

把像目标的那一部分筛出来。

例如：

```halcon
select_shape (ConnectedRegions, SelectedRegions, 'area', 'and', 500, 999999)
```

这个例子表达的是：

只保留面积在 500 到 999999 之间的区域。

除了 area，常见的筛选特征还包括：

* width
* height
* row
* column
* circularity
* rectangularity
* area_holes
* compactness

也就是说，region 的价值很大一部分就在于：

可以把目标表达成一组可筛选的形态特征。

## 六、一个最常见的基础流程示例

```halcon
read_image (Image, 'part.png')
rgb1_to_gray (Image, GrayImage)
mean_image (GrayImage, MeanImage, 5, 5)
threshold (MeanImage, Region, 120, 255)
connection (Region, ConnectedRegions)
select_shape (ConnectedRegions, SelectedRegions, 'area', 'and', 800, 999999)
fill_up (SelectedRegions, FilledRegions)
```

这条流程大概做了这些事：

1. 读图；
2. 转灰度；
3. 平滑；
4. 提区域；
5. 拆连通域；
6. 按面积筛；
7. 填充孔洞。

这已经是非常典型的一条 region 检测链。

## 七、region 后面常接哪些处理

### 1. 面积和位置统计

例如：

* 目标面积是不是超标；
* 区域中心点在哪；
* 区域数量是多少。

### 2. 形态学处理

例如：

* 去小噪点；
* 补孔洞；
* 把断裂目标连起来；
* 去掉毛刺。

### 3. 转轮廓

有些时候先用 region 分割，再把边界转成 XLD 做后续几何分析。

例如：

```halcon
gen_contour_region_xld (SelectedRegions, Contours, 'border')
```

### 4. 做缺陷判断

例如：

* 区域数量不对；
* 面积超差；
* 孔洞数量异常；
* 某类形状筛不过。

## 八、region 的优势和边界

### 优势

* 思路直观；
* 很适合大面积目标分割；
* 形态筛选很方便；
* 在缺陷检测和有无判断中非常常见。

### 边界

* 对光照和阈值比较敏感；
* 对边界精度要求很高时不一定够；
* 如果目标和背景灰度差不明显，region 流会很吃力；
* 对复杂纹理背景不一定稳。

所以不要把 region 当成万能解法，但它确实是最常见的基础入口。

## 九、几个很常见的误区

### 1. 阈值一出结果就直接判定

很多时候还没做 connection 和筛选，噪声就已经把结果带偏了。

### 2. 只按面积筛，不看其他特征

有些干扰项面积也可能接近目标，这时要增加圆度、宽高比、位置等条件。

### 3. 忽略 ROI

如果整张图上有大量无关区域，region 流很容易变得不稳。

### 4. 以为 region 不需要预处理

实际上 region 流往往比很多流程更依赖前面的预处理和背景控制。

## 十、什么时候说明你已经把 region 用对了

通常至少要满足：

1. 多张样本上能稳定提到目标区域；
2. 连通域拆分逻辑合理；
3. 筛选条件不只依赖单一面积；
4. 目标和干扰项的区分有清晰依据；
5. 你知道当前任务是不是该继续坚持 region，还是该转向 XLD 或匹配。

## 总结

1. region 是 Halcon 中最常见的目标表达方式之一；
2. 一条典型的 region 流通常是“分割 -> 连通域 -> 筛选 -> 统计”；
3. connection 和 select_shape 是 region 主线里的高频核心算子；
4. 如果目标和背景差异明显、关注的是区域特征，优先从 region 思路切入通常最自然。
