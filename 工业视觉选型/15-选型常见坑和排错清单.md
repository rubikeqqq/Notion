# 选型常见坑和排错清单

前面几篇已经把工业视觉选型的各个维度拆开讲了。

真正到项目选型阶段，最常见的问题往往不是"不知道某个参数"，而是：

- 算都算对了，但实际装上去以后发现视野不够；
- 相机选够了分辨率，但忘了算运动模糊；
- 光源买回来打光效果和想的不一样。

这篇不按单一知识点讲，而是按项目里最常见的选型坑来整理。

## 一、视野覆盖不足

### 表现

- 产品稍微偏一点，就有一部分特征在图像外面；
- 理论计算视野刚好覆盖，但安装时发现要调整。

### 根因

1. 只算了目标本身的大小，没有加产品放置公差；
2. 没有考虑产线定位误差（产品来的位置不是完全固定的）；
3. 理论 FOV 用镜头标称值算，但实际装配有偏差。

### 对策

视野计算时至少加 10-20% 的余量：

总视野 = 目标尺寸 + 定位公差 × 2 + 余量

## 二、像素当量算对了，但实际精度达不到

### 表现

- 理论上像素当量 0.02mm/pixel，但实际测量重复性差很多；
- 测量结果波动超过预期。

### 根因

1. 没有考虑亚像素定位能力（理论上限和实际稳定性的差距）；
2. 镜头畸变没有校正，边缘区域测量误差大；
3. 光源不稳定导致边缘位置偏移；
4. 运动模糊使边缘模糊，边缘检测重复性下降。

### 对策

- 像素当量只决定了理论分辨率上限，实际精度还会受光源、镜头、标定、震动等多种因素影响；
- 选型时在理论精度基础上至少留 2 倍余量；
- 精度要求高时做样机验证，不要只靠计算。

## 三、像元太小导致图像噪声大

### 表现

- 同样是 500 万像素的相机，某些型号的图像噪声明显更大；
- 光照需要很强才能获得较好的 SNR。

### 根因

同等分辨率下，传感器尺寸决定了像元大小。

小像元（< 3μm）的传感器灵敏度低，需要更强的光照才能获得好的信噪比。

### 对策

- 在满足分辨率要求的前提下，尽量选择像元更大的相机；
- 并非"像素越高越好"，过高的分辨率会牺牲灵敏度和噪声性能。

## 四、运动模糊

### 表现

- 拍高速运动的目标时图像模糊；
- 边缘无法清晰定位。

### 根因

运动模糊 = 目标运动速度 × 曝光时间

常见情景：

- 相机标称帧率足够，但曝光时间限制了帧率；
- 曝光时间设短了，但光源亮度跟不上。

### 对策

- 飞拍场景需要短曝光 + 强光源（频闪）；
- 更根本的：使用硬件触发和频闪照明冻结运动。

## 五、镜头像面尺寸和传感器不匹配

### 表现

- 图像四角发暗或出现黑角（渐晕）；
- 用了高分辨率相机，但边缘比中心模糊很多。

### 根因

镜头像面尺寸小于相机传感器尺寸，传感器边缘区域落在镜头成像圈之外。

### 对策

选镜头时确保：

镜头像面尺寸 ≥ 相机传感器尺寸

常见的像面尺寸对应关系：

- 1/2 英寸相机 → 选支持 1/2 英寸或更大的镜头；
- 2/3 英寸相机 → 选支持 2/3 英寸或更大的镜头；
- 1 英寸相机 → 选支持 1 英寸的镜头。

## 六、景深不够

### 表现

- 目标有高度差，部分区域清晰部分模糊；
- 产品高度变化时图像时而清晰时而模糊。

### 根因

高放大倍率 + 大光圈 = 极浅的景深

常见场景：微距检测、远心镜头。

### 对策

- 缩小光圈（增大 F 值）；
- 增加工作距离（但放大倍率也会变化）；
- 用自动对焦或液态镜头（成本较高）。

## 七、接口带宽不足

### 表现

- 相机标称帧率很高，实际采图帧率跑不上去；
- 偶尔丢帧；
- GigE 多相机时图像延迟。

### 根因

接口带宽 = 分辨率 × 像素位深 × 帧率

没有验算带宽就直接选接口。

### 对策

带宽验算公式：

带宽占用 = 分辨率 × 像素位深 × 帧率 × 1.2（协议开销）

确认带宽占用 ≤ 接口有效带宽的 80%（留余量）。

## 八、光源选型错误

### 表现

- 目标特征和背景在图像上对比度不够；
- 不同批次的产品打光效果差异很大；
- 反光严重，局部过曝。

### 根因

1. 没有考虑目标表面材质和反光特性；
2. 光源角度和位置选择不当；
3. 没有用实际样品做打光实验；
4. 光源颜色选择没有考虑目标颜色和吸收特性。

### 对策

- 光源选型不是选一个类型就完事的，必须做打光实验；
- 用 OK 样和 NG 样在不同光源条件下测试对比度；
- 高反光场景优先考虑同轴光、圆顶光、偏振方案。

## 九、工控机选型过于保守或激进

### 表现

- 采集卡插不上去（槽位或 PCIe 通道不够）；
- CPU 跑不满，算法执行时间超节拍；
- 工控机在现场频繁死机或过热关机。

### 根因

1. 选工控机时没有考虑 PCIe 插槽数量和类型；
2. CPU 选型只看频率，没有考虑多核性能；
3. 忽略了工业现场的极限环境温度。

### 对策

- 列清楚需要哪些扩展卡、每个什么接口，再选工控机；
- 环境温度按最恶劣情况考虑，加散热余量；
- 算法性能测试用实际相机和检测流程跑，不要只看 CPU 跑分。

## 十、没有考虑标定

### 表现

- 像素当量算了一个理论值，但实际测量物理尺寸时不准；
- 产品在视野不同位置测量结果不一致。

### 根因

选型时只算了"一个像素对应多少毫米"，但没有考虑到：

- 镜头畸变导致画面不同区域放大率不同；
- 工作距离偏差导致放大率变化；
- 没有用标定板做实际标定。

### 对策

- 尺寸测量项目必须做标定（光学标定板 + 标定流程）；
- 像素当量是理论初值，不能用它替代正式标定；
- 有畸变的镜头还需要做畸变校正。

## 十一、排错检查清单

如果选型完成后发现系统性能不满足要求，建议按这个顺序排查：

1. 视野 → 是否足够覆盖并留有余量；
2. 精度 → 像素当量是否满足要求，实际精度受限的因素是什么（光源、镜头畸变、震动还是算法）；
3. 帧率/节拍 → 瓶颈在相机帧率、接口带宽还是算法处理速度；
4. 图像质量 → 对比度是否足够，是否存在运动模糊或过曝；
5. 镜头匹配 → 像面尺寸是否匹配，焦距是否正确，畸变是否校正；
6. 光源 → 打光方案是否合理，对比度是否能区分目标特征和背景；
7. 接口/采集卡 → 带宽是否充足，是否有丢帧；
8. 控制器 → CPU 和内存是否够用，I/O 是否满足；
9. 标定 → 是否做了正式标定，结果是否准确；
10. 环境 → 温度、振动、环境光是否在系统设计范围内。

## 常用计算公式速查

| 公式 | 说明 |
|------|------|
| 像素当量 = 视野宽 ÷ 像素宽度 | 一个像素对应的物理尺寸 |
| 最小需求像素数 = 视野 ÷ 精度 × 4 | 特征覆盖 4 像素 |
| 运动模糊 = 速度 × 曝光时间 | 曝光期间目标的移动距离 |
| 带宽占用 = 分辨率 × 位深 × 帧率 | 传输数据量 |
| 放大率 M = 传感器宽 ÷ 视野宽 | 光学放大倍率 |
| f ≈ WD × M / (1 + M) | 焦距近似计算 |
| 镜头分辨率 ≥ 1000 / (2 × 像元μm) | 镜头-相机分辨率匹配 |

## 总结

1. 视野预留余量、精度留余量、带宽留余量——在工业选型中，"刚好够"往往意味着"不够"；
2. 多数精度问题的根因不在相机分辨率，而在光源、镜头畸变和标定；
3. 打光实验是选型阶段最有价值的验证手段，性价比极高；
4. 选型完成后一定要做整机联调验证，不要直接批量采购；
5. 选型出错通常不是因为不知道某个参数，而是参数之间的相互影响没有考虑到。
