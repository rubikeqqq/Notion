# 数组、字符串和结构体

前面已经有了变量、流程控制和函数，接下来最自然的问题通常是：

“如果我不只处理一个值，而是处理一组数据，该怎么组织？”

这一篇先解决三个最基础的数据组织方式：

* 数组；
* 字符串；
* 结构体。

## 一、数组是什么

数组可以先理解成：

“一组相同类型、连续存放的数据。”

最基础写法：

```cpp
int scores[5] = {90, 85, 88, 92, 76};
```

访问方式：

```cpp
std::cout << scores[0] << std::endl;
scores[1] = 95;
```

这里先记住两个特点：

1. 下标从 0 开始；
2. 越界访问不会自动被安全拦住。

这一点和很多托管环境很不一样，必须从一开始就有警惕。

## 二、字符串为什么先推荐 `std::string`

虽然 C++ 里也有 C 风格字符串，但入门阶段更推荐先用 `std::string`。

```cpp
#include <string>

std::string name = "Tom";
std::string title = "C++ Learner";
```

常见操作：

```cpp
std::string full = name + " - " + title;
std::cout << full << std::endl;
std::cout << full.size() << std::endl;
```

这样做的好处是：

* 写法更接近现代 C++；
* 不用一上来就和字符数组、结束符这些细节缠在一起；
* 后面配合 STL 会更自然。

## 三、结构体为什么值得认真学

很多人会误以为：

“有了类，结构体是不是就没用了？”

不是。

结构体在 C++ 里仍然非常常用，尤其适合：

* 表达一组相关数据；
* 轻量数据对象；
* 暂时不强调复杂行为的场景。

例如：

```cpp
#include <string>

struct Student
{
    std::string name;
    int age;
    double score;
};
```

使用方式：

```cpp
Student student;
student.name = "Tom";
student.age = 18;
student.score = 95.5;
```

## 四、数组和结构体怎么配合

一个很常见的基础练习就是：

“用结构体表示一条记录，再用数组表示一组记录。”

```cpp
Student students[3] = {
    {"Tom", 18, 95.5},
    {"Jerry", 19, 88.0},
    {"Alice", 18, 92.0}
};
```

遍历：

```cpp
for (int i = 0; i < 3; i++)
{
    std::cout << students[i].name << " " << students[i].score << std::endl;
}
```

这组写法虽然基础，但已经能很好地锻炼：

* 数组下标；
* 成员访问；
* 数据组织；
* 循环配合。

## 五、和 C# 对照时该怎么理解

### 1. 数组相似，但边界安全感不能照搬

C# 里数组访问越界通常会抛异常。

C++ 里原生数组越界属于非常危险的问题，不能靠运行时兜底。

### 2. `std::string` 更接近“推荐默认项”

它不是最底层的字符串形式，但在实际开发里通常更适合作为入门默认选择。

### 3. 结构体不只是“旧时代遗物”

在 C++ 里，结构体和类的差别没有初学者想象中那么大。

后面会继续讲它们的关系。

## 六、常见误区

### 1. 原生数组一越界就希望像 C# 一样被清楚报错

不要有这种预期。

### 2. 一上来执着于 C 风格字符串

这会把入门节奏拖慢。

### 3. 认为结构体只能放数据、完全不能有函数

这种理解太窄。

入门阶段先把它当数据组织方式理解即可，后面再细化。

## 七、小结

这一篇最重要的是先把最基础的数据组织方式理顺：

1. 数组解决“同类型的一组数据”；
2. `std::string` 解决“更现代、更好用的字符串处理”；
3. 结构体解决“多字段数据怎么组合表达”。
