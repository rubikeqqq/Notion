# 类和对象入门

前面已经把基础类型、函数、结构体、指针和引用铺过一遍了。

接下来最自然的问题通常是：

“如果我要把数据和行为封装在一起，该怎么做？”

这时就进入 C++ 的类和对象。

## 一、类解决什么问题

可以先把类理解成：

“把相关数据和相关操作打包在一起的一种组织方式。”

例如，一个学生如果只用零散变量表示：

```cpp
std::string name = "Tom";
int age = 18;
double score = 95.5;
```

程序一复杂，很快就会乱。

而类的目标就是把这些数据和围绕它的行为放在一起。

## 二、一个最基础的类长什么样

```cpp
#include <iostream>
#include <string>

class Student
{
public:
    std::string name;
    int age;

    void PrintInfo()
    {
        std::cout << name << " " << age << std::endl;
    }
};
```

使用方式：

```cpp
Student student;
student.name = "Tom";
student.age = 18;
student.PrintInfo();
```

这里先记住两个核心点：

1. `Student` 是类；
2. `student` 是对象，也叫实例。

## 三、`public`、`private` 是什么

它们是访问控制。

最常见的理解方式是：

* `public`：类外部可以访问；
* `private`：只允许类内部访问。

更常见的实际写法是：

```cpp
class Student
{
private:
    std::string name;
    int age;

public:
    void SetName(const std::string& value)
    {
        name = value;
    }

    void SetAge(int value)
    {
        age = value;
    }

    void PrintInfo() const
    {
        std::cout << name << " " << age << std::endl;
    }
};
```

这就是封装最基础的样子。

如果你从 C# 过来，最值得先建立的是这张“类的阅读顺序”小表：

| 看到类定义时，先看什么 | 为什么先看它 |
| --- | --- |
| 成员默认是不是暴露太多 | 判断封装边界是否清楚 |
| 成员函数后面有没有 `const` | 判断接口是否承诺不改对象 |
| 参数是不是大量按值传递 | 判断有没有隐藏复制成本 |
| 数据和行为是不是放在一起 | 判断这个类是不是只是数据堆积 |

## 四、结构体和类到底什么关系

这时最容易问的问题通常是：

“前一篇不是已经有结构体了吗，为什么还要类？”

在 C++ 里，结构体和类并不是完全不同的两个世界。

最核心的一个表面区别是：

* `struct` 默认成员是 `public`；
* `class` 默认成员是 `private`。

所以你可以先把它们理解成：

* 结构体更常用于偏数据表达；
* 类更常用于强调封装和行为。

## 五、成员函数是什么

类里的函数通常叫成员函数。

它和普通函数最大的区别是：

* 它属于某个类；
* 它可以直接访问当前对象的成员。

例如：

```cpp
class Counter
{
private:
    int value = 0;

public:
    void Increase()
    {
        value++;
    }

    int GetValue() const
    {
        return value;
    }
};
```

## 六、为什么成员函数后面会有 `const`

```cpp
int GetValue() const
{
    return value;
}
```

这里的 `const` 可以先理解成：

“这个函数承诺不会修改当前对象的状态。”

这是 C++ 里非常常见、也非常重要的接口表达方式。

例如下面这组写法，语义差别就很明显：

```cpp
int GetValue() const;
void SetValue(int value);
```

前者更像“读取接口”，后者更像“修改接口”。

## 七、和 C# 对照时最值得注意什么

### 1. C++ 对象可以直接按值创建

```cpp
Student student;
```

这不是“声明一个引用”，而是真的创建了一个对象。

### 2. 成员函数和普通函数的差别更值得关注语义

要关心：

* 是否修改对象；
* 是否应该加 `const`；
* 参数是否应该按引用传递。

### 3. 访问控制不仅是“面向对象语法”，还是接口设计

类到底暴露什么、不暴露什么，会直接影响后面的维护成本。

## 八、常见误区

### 1. 刚学类就把所有成员都写成 `public`

这样虽然能跑，但很容易失去封装意义。

### 2. 把类只理解成“更大的结构体”

类真正重要的是：

数据和行为一起设计，外部通过接口使用。

### 3. 完全不关注成员函数的 `const`

后面项目一大，这会让接口语义变模糊。

## 九、小结

这一篇最重要的是先把下面几件事建立起来：

1. 类是把数据和行为组织在一起的工具；
2. 对象是类的实例，可以按值直接存在；
3. `public` / `private` 是接口边界；
4. 成员函数后的 `const` 不是装饰，而是承诺。
