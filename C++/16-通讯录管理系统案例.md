# 通讯录管理系统案例

如果你刚学完基础语法、函数、数组、字符串、结构体，最适合作为第一份完整练手项目的，通常就是通讯录管理系统。

它的好处是：

* 功能不复杂；
* 很适合训练菜单流程和函数拆分；
* 能把结构体、数组或 `vector`、查找、删除这些基础能力真正用起来；
* 写完后你会第一次明显感觉到“我不是只会写语法题，而是在写一个小程序”。

## 一、这个案例最适合用来练什么

这一类项目最适合训练：

1. 结构体建模；
2. 菜单驱动流程；
3. 增删改查；
4. 函数拆分；
5. 基础的边界判断。

如果你现在对这些主题还不稳，建议先回看：

* [04-基本类型、变量和输入输出](04-基本类型、变量和输入输出.md)
* [05-流程控制和函数](05-流程控制和函数.md)
* [06-数组、字符串和结构体](06-数组、字符串和结构体.md)

## 二、先别急着写代码，先把功能拆出来

最基础的一版通讯录，通常包含这些功能就够了：

* 添加联系人；
* 显示全部联系人；
* 查找联系人；
* 修改联系人；
* 删除联系人；
* 清空通讯录；
* 退出系统。

这个阶段不要一上来就加文件存储、分页、排序、模糊搜索。

先把最基础主线写通更重要。

## 三、数据模型怎么定最合适

第一版完全可以先用结构体。

```cpp
#include <string>

struct Contact
{
    std::string name;
    std::string phone;
    std::string address;
    int age;
};
```

如果你想更贴近现代写法，可以直接配合 `std::vector<Contact>`。

```cpp
#include <vector>

std::vector<Contact> contacts;
```

这样会比手写固定长度数组更自然。

## 四、程序骨架先怎么搭

一个最小主流程通常像这样：

```cpp
int main()
{
    std::vector<Contact> contacts;
    int choice = 0;

    while (true)
    {
        ShowMenu();
        std::cin >> choice;

        switch (choice)
        {
        case 1:
            AddContact(contacts);
            break;
        case 2:
            ShowContacts(contacts);
            break;
        case 3:
            FindContact(contacts);
            break;
        case 4:
            EditContact(contacts);
            break;
        case 5:
            DeleteContact(contacts);
            break;
        case 6:
            ClearContacts(contacts);
            break;
        case 0:
            return 0;
        default:
            std::cout << "输入无效" << std::endl;
            break;
        }
    }
}
```

这个结构里最重要的不是功能多，而是流程清楚。

## 五、推荐实现顺序

如果你准备自己从零敲一遍，最稳的顺序通常是：

1. 先把 `Contact` 结构体和 `std::vector<Contact>` 准备好；
2. 先写 `ShowMenu()` 和主循环；
3. 先实现 `AddContact()` 和 `ShowContacts()`，把主流程跑通；
4. 再补 `FindContact()`；
5. 再补 `EditContact()` 和 `DeleteContact()`；
6. 最后补 `ClearContacts()` 和输入边界处理。

这个顺序的好处是：

* 每一步都能立刻验证；
* 不会一上来把函数全铺开，结果没有一个能跑；
* 先把最核心的数据流和菜单流打通。

## 六、第一版建议拆成哪些函数

如果你不想一上来设计太复杂，第一版函数骨架建议控制在下面这些：

```cpp
void ShowMenu();
void AddContact(std::vector<Contact>& contacts);
void ShowContacts(const std::vector<Contact>& contacts);
int FindContactIndex(const std::vector<Contact>& contacts, const std::string& name);
void FindContact(const std::vector<Contact>& contacts);
void EditContact(std::vector<Contact>& contacts);
void DeleteContact(std::vector<Contact>& contacts);
void ClearContacts(std::vector<Contact>& contacts);
```

这里最值得注意的是：

* “查找索引”可以单独抽函数；
* 这样删除和修改都能复用查找逻辑；
* 你会第一次明显感受到“函数拆分不是为了拆而拆，而是为了减少重复”。

## 七、一个更贴近落地的查找骨架

例如查找联系人索引可以这样写：

```cpp
int FindContactIndex(const std::vector<Contact>& contacts, const std::string& name)
{
    for (size_t index = 0; index < contacts.size(); index++)
    {
        if (contacts[index].name == name)
        {
            return static_cast<int>(index);
        }
    }

    return -1;
}
```

基于这个函数，再写删除和修改通常会顺很多。

## 八、第一版完成标准应该定到哪里

第一版不需要追求“像正式软件”，做到下面这些就够：

1. 菜单循环稳定；
2. 增删改查都能用；
3. 不依赖全局变量硬撑；
4. 查找失败时能给出清晰提示；
5. 显示联系人时输出格式至少可读。

如果这些都已经稳定，再考虑：

* 保存到文件；
* 按姓名排序；
* 模糊搜索；
* 分页显示。

## 九、函数拆分时应该怎么想

很多人第一次写这种项目，最容易犯的错误是把所有逻辑全塞进 `main`。

更稳的思路是：

* 一个菜单动作，对应一个函数；
* 修改容器的函数，参数就显式传引用；
* 只读的显示和查找函数，参数就尽量传 `const` 引用。

例如：

```cpp
void AddContact(std::vector<Contact>& contacts)
{
    Contact contact;
    std::cin >> contact.name;
    std::cin >> contact.phone;
    std::cin >> contact.address;
    std::cin >> contact.age;
    contacts.push_back(contact);
}
```

```cpp
void ShowContacts(const std::vector<Contact>& contacts)
{
    for (const auto& contact : contacts)
    {
        std::cout << contact.name << " " << contact.phone << std::endl;
    }
}
```

## 十、这个项目最值得练到什么程度

第一轮练习，做到下面这几个点就已经很有价值：

1. 不依赖全局变量把主要流程写通；
2. 会把菜单逻辑和业务逻辑拆成函数；
3. 会用结构体表达一条记录；
4. 会用 `vector` 管理一组记录；
5. 能正确处理“找不到联系人”这类最基础分支。

## 十一、常见误区

### 1. 一上来就追求“高级架构”

这一阶段重点不是设计模式，而是先把基础语法写稳。

### 2. 还没理顺函数参数，就大量依赖全局变量

这样练出来的收获会很有限。

### 3. 明明是小项目，却把所有逻辑都堆在一个超长函数里

这样看起来像做完了，实际上没有真正训练到函数拆分能力。

## 十二、小结

通讯录管理系统最适合做 C++ 的第一份完整练手项目，因为它正好能把：

1. 结构体；
2. `vector`；
3. 菜单流程；
4. 函数拆分；
5. 增删改查；

集中练一遍。
