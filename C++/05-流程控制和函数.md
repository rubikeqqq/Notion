# 流程控制和函数

学完基础类型以后，下一步最自然的问题通常是：

“程序逻辑到底怎么组织？”

这时最关键的两块就是：

* 流程控制；
* 函数。

## 一、流程控制解决什么问题

程序不是只把代码从上往下写完就结束。

你通常会遇到下面这些场景：

* 条件不同，执行不同逻辑；
* 一段逻辑要重复执行很多次；
* 某段代码应该单独封装，避免重复；
* 一部分逻辑应该对外暴露输入和输出，而不是全部写在 `main` 里。

流程控制和函数，就是先解决这些问题。

## 二、最常见的条件判断

### 1. `if`

```cpp
int score = 85;

if (score >= 90)
{
    std::cout << "优秀" << std::endl;
}
else if (score >= 60)
{
    std::cout << "及格" << std::endl;
}
else
{
    std::cout << "不及格" << std::endl;
}
```

这和 C# 很接近，但要继续保持一个习惯：

* 条件表达式最终必须明确是可判断的值；
* 大括号不要省，尤其是初学阶段。

### 2. `switch`

```cpp
int level = 2;

switch (level)
{
case 1:
    std::cout << "普通" << std::endl;
    break;
case 2:
    std::cout << "重要" << std::endl;
    break;
default:
    std::cout << "未知" << std::endl;
    break;
}
```

## 三、最常见的循环

### 1. `for`

```cpp
for (int i = 0; i < 5; i++)
{
    std::cout << i << std::endl;
}
```

### 2. `while`

```cpp
int count = 0;

while (count < 3)
{
    std::cout << count << std::endl;
    count++;
}
```

### 3. `do while`

```cpp
int input = 0;

do
{
    std::cin >> input;
} while (input <= 0);
```

它的特点是：

“先执行一次，再判断条件。”

## 四、为什么一定要学函数

很多初学者一开始会把所有逻辑堆在 `main` 里。

这在演示阶段问题不大，但只要代码稍微一长，就会立刻变得难维护。

函数的价值可以先理解成：

* 给一段逻辑起名字；
* 控制输入和输出；
* 避免重复；
* 为后面学类和模块化打基础。

## 五、一个最基础的函数长什么样

```cpp
int Add(int left, int right)
{
    return left + right;
}
```

这里可以拆成四部分：

1. 返回类型：`int`；
2. 函数名：`Add`；
3. 参数列表：`int left, int right`；
4. 函数体。

调用方式：

```cpp
int result = Add(2, 3);
```

如果你接下来准备开始写稍微像样一点的项目，这里最值得先建立的是“参数设计速查”这张小表：

| 写法 | 最常见含义 | 什么时候优先考虑 |
| --- | --- | --- |
| `void Print(User user)` | 按值传递，会产生副本 | 小对象、明确就想复制时 |
| `void Print(const User& user)` | 引用传递，不修改原对象 | 只读、大对象、高频默认项 |
| `void Update(User& user)` | 引用传递，会修改原对象 | 明确需要改调用方对象时 |
| `User Create()` | 按值返回结果对象 | 结果本身就该作为新对象返回时 |

## 六、值传递和引用传递先有一个直觉

### 1. 值传递

```cpp
void Increase(int number)
{
    number++;
}
```

如果你传进去的是一个普通 `int`，函数里操作的是它的副本。

### 2. 引用传递

```cpp
void Increase(int& number)
{
    number++;
}
```

这时操作的是原变量本身。

这部分先建立直觉就够，下一篇讲指针和引用时会系统展开。

## 七、和 C# 对照时该怎么看

### 1. 语法形式相似，不代表语义成本相同

C# 中很多传参是否复制对象，平时不太会被重点讨论。

但在 C++ 里：

* 值传递可能意味着对象拷贝；
* 引用传递意味着直接操作原对象；
* 参数设计会直接影响性能和语义清晰度。

### 2. 函数不是独立小知识，而是后面所有封装的基础

你后面学类、模板、STL，本质上都仍然在处理：

* 输入是什么；
* 输出是什么；
* 是否复制；
* 是否修改原对象。

如果你对这一点已经有感觉，后面看 [07-指针、引用和const](07-指针、引用和const.md) 时会顺很多，因为那一篇会把“参数语义”真正讲透。

## 八、常见误区

### 1. 把所有逻辑都塞进 `main`

这会让后面几乎没法扩展。

### 2. 还没理解值传递，就急着写复杂类参数

很容易在后面对象复制和生命周期问题上迷糊。

### 3. 图省事省略大括号

尤其是新手阶段，这会让阅读和排错成本升高。

## 九、小结

这一篇最重要的是把下面几件事先稳定下来：

1. 条件判断和循环负责控制程序流程；
2. 函数负责把逻辑拆开并起名字；
3. 参数设计不是随便写的，后面会直接连到对象复制和引用语义。
