# 基本类型、变量和输入输出

真正开始写 C++ 程序时，最先碰到的就是：

* 变量怎么定义；
* 类型怎么选；
* 输入输出怎么写；
* `auto`、`const` 这些关键字到底该怎么理解。

这些内容看起来基础，但会直接影响后面函数、类、指针、模板的理解。

## 一、最常见的基础类型有哪些

先记住最常见的几类：

* `int`：整数；
* `double`：双精度浮点数；
* `float`：单精度浮点数；
* `char`：字符；
* `bool`：布尔值；
* `std::string`：字符串。

最小示例：

```cpp
int age = 18;
double price = 12.5;
char grade = 'A';
bool isPassed = true;
std::string name = "Tom";
```

和 C# 的共同点很多，但要特别注意两点：

1. C++ 中字符串通常不是语言内建类型，而是标准库里的 `std::string`；
2. 类型大小和实现细节比 C# 更值得关注，但入门阶段先记常用类型即可。

如果你只想先建立“最常用类型该怎么选”的直觉，可以先记这个速查：

| 需求 | 更常见的选择 | 先这样理解 |
| --- | --- | --- |
| 普通整数 | `int` | 默认起点，最常用 |
| 小数 | `double` | 默认浮点起点，比 `float` 更常见 |
| 单个字符 | `char` | 表达单个字符，不是字符串 |
| 真 / 假状态 | `bool` | 只表达两种状态 |
| 文本 | `std::string` | 入门阶段优先于 C 风格字符串 |

## 二、变量定义最常见的几种写法

```cpp
int count = 10;
double score(95.5);
int level{3};
```

这三种都能定义变量。

如果你刚入门，可以先优先接受下面这个习惯：

* 简单类型优先用统一初始化或直接初始化；
* 不要让变量先定义、后长期不初始化。

## 三、`auto` 怎么理解

`auto` 的意思不是“弱类型”，而是“让编译器根据右侧表达式推断类型”。

```cpp
auto age = 18;
auto price = 12.5;
auto name = std::string("Tom");
```

这和 C# 的 `var` 有点像，但更适合带着这句话理解：

**你仍然是在写强类型代码，只是把明确写出类型这件事交给编译器。**

## 四、`const` 为什么这么重要

`const` 表示“这个值不希望被改”。

```cpp
const double pi = 3.1415926;
const int maxRetry = 3;
```

在 C++ 中，`const` 不只是编码风格问题，它经常直接影响：

* 函数参数是否允许修改；
* 指针和引用能否改动目标对象；
* 接口意图是否清楚。

后面到了指针和引用部分，你会发现 `const` 更重要。

## 五、输入输出怎么写

### 1. 输出

```cpp
#include <iostream>

int main()
{
    std::string name = "Tom";
    int age = 18;

    std::cout << "name = " << name << ", age = " << age << std::endl;
    return 0;
}
```

这里的 `<<` 可以先理解为“把内容送进输出流”。

### 2. 输入

```cpp
#include <iostream>

int main()
{
    int age = 0;
    std::cin >> age;
    std::cout << "age = " << age << std::endl;
    return 0;
}
```

这里的 `>>` 可以先理解为“从输入流拿数据”。

## 六、和 C# 对照时最值得注意的点

### 1. `std::cout` 不是 `Console.WriteLine`

它们都能输出，但风格不同。

C++ 的流式写法更强调运算符重载和链式拼接。

### 2. `auto` 和 `var` 接近，但不要简单划等号

两者都用于类型推断，但 C++ 的值语义、引用折叠、`const` 修饰会让推断结果更值得留意。

### 3. `const` 在 C++ 里的存在感更强

因为它和对象可变性、指针引用语义、函数签名设计都有直接关系。

## 七、这一篇最适合先形成什么习惯

如果你现在还在入门阶段，最值得先养成的是这 4 个习惯：

1. 变量尽量在定义时初始化；
2. 不要把 `auto` 理解成弱类型；
3. 字符串先优先用 `std::string`；
4. 看到 `const` 时先问“它在限制谁”。

## 八、常见误区

### 1. 认为 `auto` 会让代码“没类型”

不会。

类型仍然是确定的，只是交给编译器推导。

### 2. 一上来过分纠结每个基础类型精确占多少字节

这不是完全不重要，但入门阶段更重要的是先会选类型、会写程序、会理解值。

### 3. 把 `std::string` 当成 C 风格字符串

它们不是一回事。

`std::string` 是类，后面会继续展开。

## 九、小结

这一篇最重要的是先把三件事形成稳定直觉：

1. C++ 变量一定有明确类型；
2. `auto` 和 `const` 是高频基础能力，不是进阶技巧；
3. 输入输出基于流，不是方法调用风格。
