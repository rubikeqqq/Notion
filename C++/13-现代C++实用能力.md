# 现代 C++ 实用能力

学完类、模板和 STL 之后，下一步最应该补的不是更偏门的语法，而是日常工程里非常高频的现代 C++ 能力。

这一篇先把最常见的几块能力串起来：

* lambda；
* 范围 `for`；
* 智能指针；
* 异常；
* 文件操作。

## 一、lambda 为什么重要

lambda 可以先理解成：

“就地定义一个小函数对象。”

例如配合算法：

```cpp
std::vector<int> numbers = {1, 2, 3, 4, 5};

int count = std::count_if(numbers.begin(), numbers.end(), [](int value)
{
    return value % 2 == 0;
});
```

它的价值是：

* 局部规则就地表达；
* 少写额外辅助函数；
* 和 STL 算法搭配非常自然。

如果你只想先抓“现代 C++ 到底现代在哪”，可以先看下面这组对照：

| 更旧或更容易写乱的方式 | 更推荐先形成的写法习惯 |
| --- | --- |
| 手写很多只用一次的小辅助函数 | 用 lambda 表达局部规则 |
| 手写下标循环遍历容器 | 优先考虑范围 `for` |
| 直接裸 `new` / 裸 `delete` | 优先考虑 `make_unique` / 智能指针 |
| 到处自己拼零散资源释放 | 优先用 RAII 让对象管理资源 |

这一张表不是让你背写法，而是提醒你：现代 C++ 更强调语义清楚和资源归属清楚。

## 二、范围 `for` 为什么是高频默认写法

```cpp
for (const auto& value : numbers)
{
    std::cout << value << std::endl;
}
```

它能让遍历代码更清晰，也更少出下标错误。

## 三、智能指针为什么重要

如果你已经学过裸指针，就会知道：

* 谁分配；
* 谁释放；
* 什么时候释放；

这几件事一旦靠人工维护，很容易出问题。

智能指针就是为了让资源归属表达得更清楚。

最常见的两个：

* `std::unique_ptr`：独占所有权；
* `std::shared_ptr`：共享所有权。

例如：

```cpp
#include <memory>

auto user = std::make_unique<User>();
```

入门阶段先记住一个原则：

**能不用裸 `new`，就尽量不要直接裸 `new`。**

例如同样是创建对象：

```cpp
auto user = std::make_unique<User>();
```

通常会比下面这种更值得优先形成习惯：

```cpp
User* user = new User();
```

## 四、异常处理先理解什么

```cpp
try
{
    throw std::runtime_error("something wrong");
}
catch (const std::exception& ex)
{
    std::cout << ex.what() << std::endl;
}
```

异常不是为了把所有错误都丢出去，而是为了处理“正常流程之外的异常情况”。

## 五、文件操作为什么值得早点接触

因为很多练手项目都离不开它。

```cpp
#include <fstream>

std::ofstream output("log.txt");
output << "hello" << std::endl;
```

这类写法会直接连接前面学过的：

* 对象生命周期；
* 构造析构；
* 资源自动释放；
* 异常或状态判断。

## 六、现代 C++ 学到这个阶段，最值得形成什么习惯

1. 优先用标准库；
2. 优先写清楚所有权；
3. 优先写清楚接口是否修改对象；
4. 优先用更安全、语义更强的写法替代裸操作。

## 七、和 C# 对照时最值得注意什么

### 1. lambda 表面很像，但底层语义不完全一样

尤其是捕获列表，在 C++ 里要更明确地考虑外部变量是按值捕获还是按引用捕获。

### 2. 智能指针不是 GC

它是更明确的资源所有权模型，不是托管堆的翻版。

### 3. 文件和资源释放更值得显式关注

RAII 在这里会体现得非常明显。

## 八、常见误区

### 1. 学了智能指针就以为不用懂生命周期

不对。

智能指针建立在生命周期认知之上。

### 2. lambda 只会抄，不理解捕获

这很容易在后面出现悬垂引用或意外复制。

### 3. 例外流程和普通业务分支不区分

这会让异常设计越来越乱。

## 九、小结

这一篇不是为了把现代 C++ 全讲完，而是先把最常用、最实用的一批能力串起来：

1. lambda 负责表达局部规则；
2. 范围 `for` 负责更清晰遍历；
3. 智能指针负责更明确的资源所有权；
4. 异常和文件操作把语言能力接到工程场景。
