# 模板和泛型思维

学完类、继承和多态之后，下一步很自然会遇到一个问题：

“如果一套逻辑只是在类型上不同，我是不是要重复写很多份？”

这时模板就出现了。

## 一、模板解决什么问题

模板最核心的价值是：

“把类型也当成参数，让一份代码能适配多种类型。”

## 二、函数模板的最小例子

```cpp
template <typename T>
T Add(T left, T right)
{
    return left + right;
}
```

使用方式：

```cpp
int a = Add(1, 2);
double b = Add(1.5, 2.5);
```

这里可以先理解成：

* `T` 代表一个待定类型；
* 编译器会在具体使用时生成对应版本。

## 三、类模板的最小例子

```cpp
template <typename T>
class Box
{
public:
    void SetValue(const T& value)
    {
        data = value;
    }

    T GetValue() const
    {
        return data;
    }

private:
    T data;
};
```

使用：

```cpp
Box<int> intBox;
intBox.SetValue(10);

Box<std::string> stringBox;
stringBox.SetValue("hello");
```

## 四、它和 C# 泛型像在哪里，不像在哪里

像的地方：

* 都想解决代码复用；
* 都允许把类型参数化；
* 都能减少重复实现。

不太一样的地方：

* C# 泛型更多依赖运行时和语言运行环境；
* C++ 模板更偏编译期展开；
* 模板报错往往更长、更绕。

所以学模板时，一定要更关注“编译期生成了什么”这层直觉。

## 五、模板真正重要的地方不只是语法

模板背后的核心思维是：

**如果一段逻辑对多种类型都成立，那就尽量把差异抽成类型参数，而不是复制代码。**

这也是 STL 的基础。

## 六、为什么很多人学模板会觉得难

因为它同时要求你具备三层能力：

1. 对函数和类已经熟悉；
2. 对类型参数化有直觉；
3. 对编译期错误能保持耐心。

所以模板不适合太早学，但到了这个阶段就必须开始接触。

## 七、常见误区

### 1. 把模板只当成“更难的泛型语法”

这会忽略它真正的编译期意义。

### 2. 一上来就钻进非常复杂的模板技巧

没必要。

先把函数模板、类模板和 STL 使用建立起来更重要。

### 3. 模板报错一长就完全放弃读信息

应该先抓第一处真正的类型不匹配点，而不是被整段报错吓住。

## 八、小结

这一篇最重要的是建立泛型思维：

1. 模板让类型也能成为参数；
2. 它的核心目标是复用，而不是炫技；
3. STL 大量能力都建立在模板之上。
