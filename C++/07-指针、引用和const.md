# 指针、引用和 const

如果说前面几篇还属于“很多语言都有的基础”，那么从这一篇开始，才真正进入 C++ 的核心地带。

很多 C# 开发者学 C++ 卡住，通常就卡在这里：

* 指针到底是什么；
* 引用和指针有什么区别；
* `const` 到底修饰谁；
* 为什么这些东西会影响后面几乎所有主题。

## 一、先理解地址是什么

每个变量在内存里通常都会有一个位置。

你可以先把“地址”理解成这个位置的编号。

```cpp
int age = 18;
std::cout << &age << std::endl;
```

这里的 `&age` 表示“取 age 的地址”。

## 二、指针是什么

指针就是“专门用来保存地址的变量”。

```cpp
int age = 18;
int* ptr = &age;
```

这里：

* `age` 是一个 `int`；
* `&age` 是它的地址；
* `ptr` 是一个指向 `int` 的指针。

如果想通过指针访问它指向的值，可以解引用：

```cpp
std::cout << *ptr << std::endl;
*ptr = 20;
std::cout << age << std::endl;
```

## 三、引用是什么

引用可以先理解成“某个对象的别名”。

```cpp
int age = 18;
int& ref = age;
ref = 20;
std::cout << age << std::endl;
```

这里 `ref` 不是一个新的 `int`，而是 `age` 的别名。

所以引用和指针的直观差别可以先记成：

* 指针保存地址，可以改指向，也可以为空；
* 引用是别名，一旦绑定通常不再改绑，也不能是空引用。

如果你只想先快速分清这三个概念，可以先记这个速查：

| 形式 | 最先这样理解 | 最值得先注意什么 |
| --- | --- | --- |
| `int value` | 一个普通值对象 | 按值使用、按值复制 |
| `int* ptr` | 一个保存地址的变量 | 是否为空、是否指向有效对象 |
| `int& ref` | 一个已有对象的别名 | 一旦绑定通常不再改绑 |

## 四、为什么说 C# 的“引用类型”不能直接照搬过来

在 C# 里，“引用类型”更多是在说对象的存储和访问语义。

而在 C++ 里：

* 引用是语言级别的别名机制；
* 指针是显式地址变量；
* 值对象、指针、引用是三种需要明确区分的东西。

所以学 C++ 时，必须把这三个概念拆开，而不是用 C# 的一句“这是引用类型”混过去。

## 五、`const` 到底怎么读

### 1. 最基础的 const 变量

```cpp
const int maxRetry = 3;
```

表示 `maxRetry` 不能再被修改。

### 2. const 引用

```cpp
const std::string& name = userName;
```

可以先理解成：

“我通过引用访问这个对象，但不允许通过这个引用修改它。”

这在函数参数中非常常见。

### 3. const 指针和指向 const 的指针

这是最容易绕的地方。

```cpp
const int* p1 = &value;
int* const p2 = &value;
```

可以先这样读：

* `const int* p1`：不能通过 `p1` 改值；
* `int* const p2`：`p2` 自己不能改指向。

入门阶段先记住“`const` 修饰谁，就限制谁”这个原则就够了。

## 六、函数里为什么大量出现引用和 const

因为它们同时解决两件事：

1. 避免不必要的对象复制；
2. 明确告诉调用方“我会不会改你的对象”。

例如：

```cpp
void PrintName(const std::string& name)
{
    std::cout << name << std::endl;
}
```

这段写法的意思是：

* 不复制 `name`；
* 不修改 `name`；
* 只是读取它。

这是现代 C++ 很高频的基础写法。

如果你在函数参数这里总是拿不准，可以先用这张小表判断：

| 写法 | 常见意图 |
| --- | --- |
| `User user` | 我接受复制成本，或者对象本来就很小 |
| `User& user` | 我就是要改原对象 |
| `const User& user` | 我不想复制，也不想改原对象 |

## 七、最常见的危险点是什么

### 1. 野指针

指针没有指向有效对象，却被拿来访问。

### 2. 空指针解引用

```cpp
int* ptr = nullptr;
*ptr = 10;
```

这是典型危险写法。

### 3. 返回局部变量地址或引用

```cpp
int* GetPointer()
{
    int value = 10;
    return &value;
}
```

局部变量离开作用域就销毁，这样返回出去的地址是无效的。

## 八、和 C# 对照时该抓什么

如果你是从 C# 转过来的，这一篇最值得建立的不是“语法熟练度”，而是下面这几层判断：

1. 这个变量是值、指针还是引用；
2. 这次传参有没有复制；
3. 这段代码有没有可能访问无效地址；
4. `const` 到底是在约束对象，还是约束指针本身。
