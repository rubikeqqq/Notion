# STL 容器和算法入门

如果说模板让你看到“代码怎么泛型化”，那么 STL 就是这个思想在标准库里的大规模落地。

它是现代 C++ 最重要的工具箱之一。

## 一、先理解 STL 解决什么问题

STL 不是单一组件，而是一套组合思路。

它通常包括：

* 容器：存数据；
* 迭代器：访问数据；
* 算法：处理数据；
* 函数对象和谓词：描述规则。

## 二、最常用的几个容器先记住什么

如果你现在还没有容器选型直觉，先用这张小速查就够了：

| 场景 | 更常见的起点 | 先这样理解 |
| --- | --- | --- |
| 一组顺序数据，最常见默认项 | `vector` | 最像“动态数组” |
| 键值映射 | `map` | 根据 key 找 value |
| 去重、判存在 | `set` | 自动保持唯一 |

先把这三个用稳，比一开始背完整个 STL 容器家族更重要。

### 1. `vector`

最常用、最像“默认动态数组”的容器。

```cpp
#include <vector>

std::vector<int> numbers = {1, 2, 3, 4};
numbers.push_back(5);
```

如果你不确定先用什么容器，很多场景先想到 `vector` 是合理的起点。

### 2. `map`

适合做键值对映射。

```cpp
#include <map>

std::map<std::string, int> scores;
scores["Tom"] = 95;
```

### 3. `set`

适合做去重或判定元素是否存在。

```cpp
#include <set>

std::set<int> ids = {1, 2, 3};
```

## 三、什么是迭代器

可以先把迭代器理解成：

“一种统一访问容器元素的位置对象。”

例如：

```cpp
for (auto it = numbers.begin(); it != numbers.end(); ++it)
{
    std::cout << *it << std::endl;
}
```

虽然现代 C++ 常用范围 `for`，但理解迭代器仍然很重要，因为很多算法都建立在它上面。

## 四、最常见的算法先会哪几个

### 1. `sort`

```cpp
#include <algorithm>

std::sort(numbers.begin(), numbers.end());
```

### 2. `find`

```cpp
auto it = std::find(numbers.begin(), numbers.end(), 3);
if (it != numbers.end())
{
    std::cout << "found" << std::endl;
}
```

STL 的一个重要思路就是：

**尽量把“容器是什么”和“操作规则是什么”分开。**

如果你是从 C# 过来的，可以先用一个很粗的对照理解：

* `vector` 的高频使用感，接近 `List<T>`；
* `map` 的高频使用感，接近 `Dictionary<TKey, TValue>`；
* STL 算法和容器的配合思路，和“集合 + LINQ/规则”有一点相似，但接口风格不同。

## 五、为什么 STL 比手写数组更值得学

因为它通常能带来：

* 更清晰的语义；
* 更少的边界错误；
* 更高的代码复用；
* 更接近现代 C++ 的主流写法。

这不代表原生数组没用了，而是大多数工程场景不应该先从最底层轮子开始写。

## 六、和 C# 对照时该怎么理解

### 1. `vector` 很像 `List<T>` 的高频使用感

但底层语义、扩容、迭代器失效等问题仍然要更敏感。

### 2. STL 算法更强调“算法和数据结构解耦”

这和 C# LINQ 的部分思路有点相似，但接口风格不同。

### 3. 容器选择会直接影响性能特征

这一点在 C++ 工程里通常更值得被显式考虑。

## 七、常见误区

### 1. 一上来就想把所有容器全部记住

没必要。

先把 `vector`、`map`、`set`、常见算法用稳更重要。

### 2. 只会 `for` 循环，不去接触标准算法

这样会错过 STL 最大的价值。

### 3. 不理解迭代器失效

这会在后面修改容器内容时造成很难排查的问题。

## 八、小结

这一篇最重要的是先把 STL 理解成一套组合系统：

1. 容器负责存数据；
2. 迭代器负责访问；
3. 算法负责处理；
4. 标准库优先于重复造轮子。
