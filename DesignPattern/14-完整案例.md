# 完整案例：工控视觉检测系统中的设计模式运用

前面把常用设计模式分篇讲完了。

这一篇通过一个具体的工控视觉检测项目，把多个模式组合起来运用。

## 一、项目背景

一个手机中框尺寸检测系统，支持多种产品型号，需要集成多个相机品牌，检测结果需要上报 MES 系统。

### 系统需求

- 支持 3 种手机型号切换（A、B、C）；
- 每个型号的检测流程不同（测量项不同、精度不同）；
- 相机可能是 GigE 或 USB3 接口，品牌可能是 Basler 或海康；
- 检测完成后需要：更新界面 → 保存日志 → 上传 MES → 发送 OK/NG 给 PLC。

## 二、架构总览

这个系统使用了 6 种设计模式的组合：

| 模式 | 位置 | 解决什么问题 |
|------|------|-------------|
| 外观模式 | InspectionStation | 对外暴露统一的检测接口 |
| 策略模式 | 检测算法 | 不同产品型号的检测逻辑切换 |
| 工厂模式 | 创建策略 | 根据型号创建对应的检测策略 |
| 适配器模式 | 相机集成 | 统一不同品牌相机的接口 |
| 观察者模式 | 结果通知 | 检测完成后通知多个处理模块 |
| 模板方法 | 检测流程 | 标准化检测流程骨架 |

## 三、各层实现

### 1. 适配器层：统一相机接口

```csharp
// 项目内部统一的相机接口
public interface ICamera
{
    ImageData Acquire();
    void Start();
    void Stop();
    void SetExposure(double us);
}

// Basler 相机适配器
public class BaslerCameraAdapter : ICamera
{
    private readonly BaslerCamera _basler = new BaslerCamera();
    
    public ImageData Acquire()
    {
        var frame = _basler.Snap();
        return ConvertToImageData(frame);
    }
    
    public void Start() => _basler.Connect();
    public void Stop() => _basler.Disconnect();
    public void SetExposure(double us) => _basler.ExposureTime = us;
}

// 海康相机适配器  
public class HikCameraAdapter : ICamera
{
    private readonly HikCamera _hik = new HikCamera();
    
    public ImageData Acquire()
    {
        var raw = _hik.GetFrame();
        return ConvertToImageData(raw);
    }
    
    public void Start() => _hik.Open();
    public void Stop() => _hik.Close();
    public void SetExposure(double us) => _hik.SetExposureTime(us);
}
```

```csharp
public abstract class BaseInspector : IProductInspector
{
    public InspectionResult Inspect(ImageData image)
    {
        Preprocess(image);
        var pose = Match(image);
        var measures = Measure(image, pose);
        var result = CreateResult(measures);
        OnCompleted(result);
        return result;
    }
    
    protected virtual void Preprocess(ImageData image) { }
    protected abstract MatchResult Match(ImageData image);
    protected abstract List<Measurement> Measure(ImageData image, MatchResult pose);
    protected virtual InspectionResult CreateResult(List<Measurement> measures) { /* ... */ }
    protected virtual void OnCompleted(InspectionResult result) { }
}
```

### 6. 外观层：统一对外接口

```csharp
public class InspectionStation
{
    private readonly ICamera _camera;
    private readonly IProductInspector _inspector;
    private readonly InspectionResultPublisher _publisher;
    
    public InspectionStation(
        ICamera camera, 
        IProductInspector inspector,
        InspectionResultPublisher publisher)
    {
        _camera = camera;
        _inspector = inspector;
        _publisher = publisher;
    }
    
    public InspectionResult RunOnce()
    {
        var image = _camera.Acquire();
        var result = _inspector.Inspect(image);
        
        // 发布检测结果（观察者模式自动通知所有订阅者）
        _publisher.Publish(result);
        
        return result;
    }
}
```

## 四、系统初始化

```csharp
// DI 容器注册（以 IServiceCollection 为例）
var services = new ServiceCollection();

// 相机——使用适配器模式，根据配置选择具体实现
services.AddSingleton<ICamera>(sp =>
{
    var config = sp.GetRequiredService<ConfigManager>();
    return config.CameraBrand switch
    {
        "Basler" => new BaslerCameraAdapter(),
        "Hik" => new HikCameraAdapter(),
        _ => throw new NotSupportedException()
    };
});

// 检测策略——使用工厂模式
services.AddSingleton<InspectionStrategyFactory>();
services.AddTransient<IProductInspector>(sp =>
{
    var factory = sp.GetRequiredService<InspectionStrategyFactory>();
    var config = sp.GetRequiredService<ConfigManager>();
    return factory.Create(config.ProductType);
});

// 观察者——注册订阅者
services.AddSingleton<InspectionResultPublisher>();
services.AddSingleton<UiUpdater>();
services.AddSingleton<MesUploader>();
services.AddSingleton<PlcSender>();

// 外观——对外统一入口
services.AddSingleton<InspectionStation>();

var provider = services.BuildServiceProvider();

// 启动
var station = provider.GetRequiredService<InspectionStation>();
var result = station.RunOnce();
```

## 五、扩展场景

### 要加新产品型号

1. 新建一个 Inspector 类继承 BaseInspector（或直接实现 IProductInspector）；
2. 在 InspectionStrategyFactory 中注册新产品类型。

不需要修改任何已有代码——符合开闭原则。

### 要换相机品牌

1. 新建一个 Adapter 类实现 ICamera；
2. 修改配置中的相机品牌。

不需要修改检测流程代码——适配器模式提供了隔离层。

### 要增加 NG 报警功能

1. 新建一个 AlarmObserver，订阅 InspectionCompleted 事件；
2. 在 NG 时触发报警。

不需要修改任何已有代码——观察者模式提供了扩展点。

## 总结

1. 在实际的工控视觉项目中，多个设计模式通常是组合使用的；
2. 适配器模式隔离了第三方 SDK 的变化，策略模式隔离了业务逻辑的变化，观察者模式隔离了通知处理的变化；
3. 外观模式提供了统一的调用入口，工厂模式负责创建策略，模板方法标准化了流程骨架；
4. 设计模式组合使用的关键在于：每个模式解决自己那一层的关注点，各层之间通过接口解耦；
5. 良好的架构设计让系统在面临需求变化时，只需要增加新代码而不是修改已有代码。
