# 关系型数据库基础概念

在深入学习具体数据库之前，先把关系型数据库最核心的基础概念梳理一遍。

这些概念不管用 SQL Server、SQLite 还是 MySQL 都是一样的。理解了这些，后面的内容才会顺。

## 一、表（Table）

数据库中最基础的存储单元。一个表代表一类实体的集合。

例如在工控项目中：

```sql
CREATE TABLE InspectionResults (
    Id          INT PRIMARY KEY,
    ProductName NVARCHAR(50)    NOT NULL,
    BatchNo     NVARCHAR(30),
    IsPassed    BIT             NOT NULL,
    InspectTime DATETIME2       NOT NULL,
    Operator    NVARCHAR(20)
);
```

每个表由行和列组成：

- 行（Row）代表一条记录，比如一次检测的结果；
- 列（Column）代表一个属性，比如检测时间、是否合格。

## 二、主键（Primary Key）

主键是表中唯一标识每一行数据的列或列组合。

核心要求：

- 唯一性：不能有重复值；
- 非空：不能为 NULL。

```sql
-- 单列主键
CREATE TABLE Products (
    ProductId INT PRIMARY KEY,
    Name NVARCHAR(50)
);

-- 复合主键（多列联合唯一）
CREATE TABLE InspectionLog (
    ProductId   INT,
    InspectTime DATETIME,
    PRIMARY KEY (ProductId, InspectTime)
);
```

工控项目中常见的主键策略：

- 自增整数（INT IDENTITY）：简单，适合大部分场景；
- GUID（UNIQUEIDENTIFIER）：适合分布式系统合并数据；
- 业务主键（如产品条码）：查询方便，但改动成本高。

## 三、外键（Foreign Key）

外键用来维护两个表之间的引用完整性。它确保子表中的值必须在父表中存在。

```sql
CREATE TABLE MeasurementResults (
    Id          INT PRIMARY KEY,
    InspectId   INT FOREIGN KEY REFERENCES InspectionResults(Id),
    ItemName    NVARCHAR(50),
    Value       FLOAT,
    Tolerance   FLOAT
);
```

外键的作用：防止出现"引用了不存在的数据"。

在工控项目中，外键通常在表设计时就定义好，但在高并发写入场景中为了性能有时会去掉外键，靠应用程序保证数据完整性。

## 四、索引（Index）

索引的作用是加快数据查询速度。

```sql
-- 创建索引
CREATE INDEX IX_InspectionResults_ProductName 
ON InspectionResults(ProductName);
```

### 索引的核心原理

索引类似于书的目录——没有索引要逐页翻（全表扫描），有索引可以直接定位。

### 工控场景中的索引策略

- 经常按时间范围查 → 给时间列加索引；
- 经常按产品型号筛选 → 给型号列加索引；
- 经常用来 JOIN 的列 → 加索引；
- 写入频繁的表 → 索引不宜过多（索引拖慢写入）。

## 五、事务（Transaction）

事务是将一组操作打包成一个"要么全做，要么全不做"的单元。

### ACID 四大特性

| 特性 | 说明 | 通俗理解 |
| ------ | ------ | ---------- |
| 原子性（Atomicity） | 事务内操作全成功或全回滚 | 不能只做一半 |
| 一致性（Consistency） | 事务前后数据状态一致 | 数据不会变乱 |
| 隔离性（Isolation） | 并发事务互不干扰 | 互不影响 |
| 持久性（Durability） | 提交后数据不会丢 | 不会白做 |

### C# 中使用事务

```csharp
using var transaction = connection.BeginTransaction();
try
{
    // 多个操作
    command1.Transaction = transaction;
    command1.ExecuteNonQuery();
    
    command2.Transaction = transaction;
    command2.ExecuteNonQuery();
    
    transaction.Commit();
}
catch
{
    transaction.Rollback();
    throw;
}
```

### 工控中的事务场景

- 保存检测结果时同时更新产品统计数量——要么都成功要么都回滚；
- 批量写入多个测量项时——保证一批数据要么全写入要么全不写。

## 六、视图（View）

视图是一个虚拟表，它封装了一个查询。

```sql
CREATE VIEW V_RecentDefects AS
SELECT ProductName, DefectType, InspectTime
FROM InspectionResults
WHERE IsPassed = 0
  AND InspectTime > DATEADD(DAY, -7, GETDATE());
```

视图的作用：

- 简化复杂查询（把经常用的 JOIN 封装成视图）；
- 隐藏底层表结构（查询层只接触视图，不直接访问表）。

## 七、存储过程（Stored Procedure）

存储过程是预编译在数据库中的一组 SQL 语句。

```sql
CREATE PROCEDURE usp_GetProductStats
    @ProductName NVARCHAR(50)
AS
BEGIN
    SELECT 
        COUNT(*) AS Total,
        SUM(CASE WHEN IsPassed = 0 THEN 1 ELSE 0 END) AS DefectCount
    FROM InspectionResults
    WHERE ProductName = @ProductName;
END;
```

在 C# 中调用：

```csharp
using var cmd = new SqlCommand("usp_GetProductStats", connection);
cmd.CommandType = CommandType.StoredProcedure;
cmd.Parameters.AddWithValue("@ProductName", "ProductA");
```

SQLite 不支持存储过程（可以用用户函数替代），SQL Server 和 MySQL 都支持。

## 八、NULL

NULL 表示"未知"或"没有值"，不等于 0 或空字符串。

使用 NULL 的注意事项：

```sql
-- NULL 的判断不能用 =，要用 IS NULL
SELECT * FROM InspectionResults WHERE Operator IS NULL;

-- NULL 在计算中会传播
SELECT NULL + 5; -- 结果是 NULL

-- COALESCE 处理 NULL 的默认值
SELECT COALESCE(Operator, 'Unknown') FROM InspectionResults;
```

在表设计中，应把"可选"的列设为允许 NULL，必须填的列设为 NOT NULL。

## 九、数据类型

关系型数据库中常见的数据类型：

| 类别 | SQL Server | SQLite | MySQL |
| ------ | ------------ | -------- | ------- |
| 整数 | INT, BIGINT | INTEGER | INT, BIGINT |
| 小数 | FLOAT, DECIMAL | REAL | FLOAT, DECIMAL |
| 字符串 | NVARCHAR, VARCHAR | TEXT | VARCHAR, TEXT |
| 日期时间 | DATETIME2 | TEXT(ISO) | DATETIME |
| 布尔 | BIT | INTEGER(0/1) | TINYINT |
| 二进制 | VARBINARY | BLOB | BLOB |

设计表时选择合适的数据类型可以减少存储空间、提高查询效率。

## 总结

1. 表是关系型数据库最基本的存储单元，由行和列组成；
2. 主键唯一标识每一行，外键保证数据引用的完整性；
3. 索引加速查询但会拖慢写入，需要在性能和写入之间权衡；
4. 事务保证一组操作要么全做要么全不做，ACID 是事务的核心特性；
5. 视图和存储过程可以简化查询逻辑、提高代码复用性；
6. NULL 表示"无值"，在判断和计算时需要特殊处理。
