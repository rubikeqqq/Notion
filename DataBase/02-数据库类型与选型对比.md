# 数据库类型与选型对比

在工控项目中选数据库，经常会面临一个问题：SQL Server、SQLite、MySQL 甚至一些 NoSQL 数据库摆在面前，到底选哪个。

这篇把常见的数据库类型和选型原则放在一起对比。

## 一、先说结论

如果只给一个结论，可以这样理解：

- SQL Server：Windows 生态深度整合，适合中大型项目，需要部署数据库服务；
- SQLite：零配置嵌入式数据库，适合本地数据缓存、单机应用；
- MySQL：开源跨平台，适合独立服务器部署的 Web 或数据服务场景；
- NoSQL（如 MongoDB、InfluxDB）：适合非结构化数据或时序数据。

在工控 WPF 项目中，最常见的组合是 SQLite 做本地缓存 + SQL Server 做远程数据汇总。

## 二、关系型数据库 vs NoSQL

### 关系型数据库

特点：

- 数据以表形式组织，有严格的 schema；
- 支持 ACID 事务；
- 使用 SQL 查询；
- 适合结构化数据。

工控项目里凡是检测结果、产品记录、设备参数这类结构明确的数据，都用关系型数据库。

### NoSQL 数据库

常见分类：

- 文档型（MongoDB）：适合配置信息、JSON 格式的检测报告；
- 时序型（InfluxDB、TimescaleDB）：适合设备温度、振动等连续采集的数据；
- 键值型（Redis）：适合缓存、实时状态。

在工控项目中，NoSQL 通常作为关系型数据库的补充，而不是替代。比如用 InfluxDB 存设备温度趋势数据，用关系型数据库存检测结果。

## 三、三大关系型数据库横向对比

| 特性 | SQL Server | SQLite | MySQL |
| ------ | ------------ | -------- | ------- |
| 部署方式 | 客户端-服务器 | 嵌入式（文件型） | 客户端-服务器 |
| 安装复杂度 | 需要安装服务 | 零配置（DLL 引用） | 需要安装服务 |
| 操作系统 | Windows（Linux 有限） | 全平台 | Windows/Linux |
| 数据库大小 | 无上限（企业版） | 单文件最大约 140TB | 无上限 |
| 并发写入 | 支持高并发 | 写并发受限（WAL 模式改善） | 支持高并发 |
| .NET 生态 | SqlClient（微软官方） | Microsoft.Data.Sqlite | MySqlConnector / Oracle 官方 |
| EF Core 支持 | 第一梯队 | 第一梯队 | 第一梯队 |
| 事务 | 完全支持 | 支持 | 完全支持 |
| 存储过程/函数 | 支持（T-SQL） | 有限支持 | 支持 |
| 用户权限管理 | 完善 | 不支持（文件权限） | 完善 |
| 备份方式 | 备份工具/命令 | 直接复制文件 | mysqldump / 工具 |
| 许可证 | 商业（有免费版） | 公共领域 | GPL（有商业版） |

## 四、工控场景的选型原则

### 1. 看部署环境

- 项目运行在独立的 Windows 工控机上，不需要联网上传 → SQLite
- 项目需要多台工控机共享数据，有服务器 → SQL Server
- 项目需要跨平台，或者 web 端需要访问 → MySQL
- 三者混搭：本地 SQLite 缓存 + 远程 SQL Server 或 MySQL

### 2. 看数据量级

- 日均写入几千条，单机运行 → SQLite 足够
- 日均写入几十万条，需要数据分析 → SQL Server 或 MySQL
- 需要存长时间序列数据 → 考虑时序数据库

### 3. 看并发需求

- 单进程访问 → SQLite 完全够用
- 多进程或多客户端并发写 → SQL Server 或 MySQL
- 高并发读少写 → 都可以

### 4. 看团队能力

- .NET 团队，Windows 生态 → SQL Server 是最自然的选择
- 团队熟悉开源技术栈 → MySQL
- 只想写代码不想管数据库运维 → SQLite

## 五、一个更贴近工控项目的理解方式

### 单机检测设备

工控机运行 WPF 检测软件，存储检测结果和日志：

→ SQLite 是最合适的选择。零配置、文件型、不需要装数据库服务。

### 多工位产线

多台检测工位，需要将结果统一汇总到服务器：

→ 每个工位本地 SQLite 缓存 + 服务器 SQL Server 汇总上报。

### 大型 MES 对接

工控软件需要和工厂 MES 系统交互，存储产品追溯数据：

→ SQL Server 或 MySQL 作为主数据库，通过服务层提供数据接口。

## 六、学习数据库时最容易犯的理解偏差

1. 把 SQLite 当成"简化版 SQL Server"来用，忽略了它的并发限制；
2. 小项目也用 SQL Server，增加了部署和维护成本；
3. 不区分数据库的 SQL 方言，跨数据库迁移时报语法错误；
4. 在工控场景中把检测结果一条条 INSERT 而不是批量写入，性能差。

## 总结

1. 关系型数据库是工控项目的主力，NoSQL 作为补充；
2. 三大数据库各有侧重：SQL Server 适合 Windows 中大型项目，SQLite 适合本地嵌入式，MySQL 适合开源跨平台；
3. SQLite 本地缓存 + SQL Server 远程汇总是工控项目里很常见的组合；
4. 选型时重点看部署环境、数据量级、并发需求和团队能力。
