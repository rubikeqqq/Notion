# 数据库总览

这一组笔记主要围绕关系型数据库在 .NET / WPF 工控项目中的使用展开。

数据库在工控软件中扮演的角色和普通业务系统不太一样。它不只是"存数据的地方"，更关键的是：

- 产线检测结果要存、要查、要追溯；
- 设备运行状态要记录、要分析、要报警；
- 本地缓存和远程数据库可能要协同工作；
- 采集频率高、写入量大、查询要求快。

所以数据库选型不只是选一个"用哪个"，还要考虑工控场景的写入模式、查询模式、数据量级和可靠性要求。

## 一、这组笔记主要解决什么

如果把数据库放到工控项目里理解，这组笔记主要解决四件事：

1. 如何在工控场景中选择合适的数据库（SQL Server 还是 SQLite 还是 MySQL）；
2. 如何在 C# / WPF 项目中连接和操作数据库；
3. 如何设计适合产线数据存储的表结构；
4. 数据库使用中常见的坑和排查方法。

## 总结

1. 数据库在工控项目中不只是存数据，还要满足写入频率、查询速度、数据追溯等特定要求；
2. 这组内容按"认知→基础→实践→工控→具体数据库→排错"六层组织；
3. 先理解通用的数据库概念和 SQL 语法，再深入具体数据库的差异；
4. 这组笔记的目标不是成为 DBA，而是能在 C# 工控项目中正确使用数据库。
